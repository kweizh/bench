# Wasp Task Priority Application

## Background
Wasp is a declarative DSL for building full-stack web applications. In this task, you will build a Task List application that allows users to set and toggle the priority of their tasks.

## Requirements
- **Data Model**:
    - `Task` entity with `description` (String) and `priority` (String, default: "Low").
- **Operations**:
    - `getTasks` Query: Fetch all tasks from the database.
    - `createTask` Action: Create a new task with a given description and default priority.
    - `togglePriority` Action: Toggle the `priority` of a task between "High" and "Low".
- **Frontend**:
    - A main page that displays the list of tasks.
    - A form to add new tasks.
    - For each task, show its priority and a "Toggle Priority" button.
    - Use Wasp's `useQuery` and `useAction` hooks.

## Implementation Guide
1. **Initialize Project**:
   - Create a new Wasp project in `/home/user/task-priority` using the minimal template: `wasp new task-priority -t minimal`.
2. **Define Schema**:
   - In `schema.prisma`, define the `Task` model with `description` and `priority`.
3. **Configure App**:
   - In `main.wasp`, define the `route` and `page` for `MainPage`.
4. **Implement Operations**:
   - Declare `query getTasks`, `action createTask`, and `action togglePriority` in `main.wasp`.
   - Implement the logic in `src/queries.ts` and `src/actions.ts`.
5. **Build UI**:
   - Use `useQuery` and `useAction` hooks in `src/MainPage.tsx` to interact with the backend.
6. **Database Migration**:
   - Run `wasp db migrate-dev` to apply your schema changes.

## Constraints
- **Project Path**: `/home/user/task-priority`
- **Start Command**: `wasp start`
- **Port**: 3000
- **Database**: SQLite (default)
