## Key Design Decisions

### `models.js`
- **SQLite** dialect with `database.sqlite` as the storage file, logging disabled.
- Three models defined without `timestamps` to keep the schema lean:
  - `User` — `username` (unique string)
  - `Project` — `name` (unique string)
  - `UserProject` — junction table with a `role` string column
- **`belongsToMany`** declared on both sides, pointing `through: UserProject` with explicit `foreignKey`/`otherKey` so Sequelize correctly maps `UserId` → `ProjectId`.

### `cli.js`
- `sequelize.sync()` is called first to ensure tables exist before any query runs.
- **`add`**: Uses `findOrCreate` for both `User` and `Project` (idempotent), then `UserProject.upsert()` to set/update the role for that pair.
- **`list`**: Uses the generated `user.getProjects()` association helper with `joinTableAttributes: ['role']` so the junction data is loaded; the result is mapped to `{ name, role }` objects and printed as formatted JSON.

## Usage

```bash

# List all projects for a user

# → [{ "name": "Alpha Project", "role": "developer" }]

# Non-existent user returns empty array
node cli.js list nobody
# → []
```   
