## Key design decisions

### Models (`paranoid: true`)
Both `User` and `Post` are defined with `paranoid: true`, which makes Sequelize add a `deletedAt` timestamp column and automatically filter out soft-deleted rows in normal queries.

### Cascading via Sequelize hooks
Two hooks are registered on the `User` model:

| Hook | Action |
|---|---|
| `afterDestroy` | Calls `Post.destroy({ where: { userId } })` — soft-deletes all posts belonging to the user |
| `afterRestore` | Calls `Post.restore({ where: { userId }, paranoid: false })` — restores all soft-deleted posts; `paranoid: false` is required so Sequelize looks in the deleted rows too |

### API Endpoints
| Method | Path | Description | Status |
|---|---|---|---|
| POST | `/users` | Create user | 201 |
| POST | `/users/:id/posts` | Create post for user | 201 |
| DELETE | `/users/:id` | Soft-delete user + cascade to posts | 200 |
| POST | `/users/:id/restore` | Restore user + cascade restore posts | 200 |
| GET | `/posts/:id` | Fetch post (404 if soft-deleted) | 200/404 |
