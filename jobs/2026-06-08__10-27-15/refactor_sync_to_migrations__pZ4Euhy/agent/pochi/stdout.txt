# Refactor Sequelize Sync to Migrations ## Background In development, developers often use `sequelize.sync({ force: true })` to quickly apply model changes to the database. However, this is highly dangerous in production as it drops and recreates tables, leading to data loss. The best practice is to use `sequelize-cli` to manage database schema changes via migrations. You have an existing Express application that uses Sequelize with SQLite.
