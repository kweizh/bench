## Key implementation details

### Polymorphic associations (`index.js`)
```
User  в”Ђв”ЂhasManyв”Ђв”Ђв–¶  Image  (scope: { imageableType: 'user' },    as: 'profilePictures')
Product в”Ђв”ЂhasManyв”Ђв”Ђв–¶ Image  (scope: { imageableType: 'product' }, as: 'productPhotos')
Image в”Ђв”ЂbelongsToв”Ђв”Ђв–¶ User    (foreignKey: imageableId, constraints: false, as: 'user')
Image в”Ђв”ЂbelongsToв”Ђв”Ђв–¶ Product (foreignKey: imageableId, constraints: false, as: 'product')
```

- **`constraints: false`** вЂ” required because `imageableId` is a shared FK that points to different tables; SQLite can't enforce a normal FK constraint on it.
- **`scope`** on `hasMany` вЂ” Sequelize automatically appends `WHERE imageableType = 'user'` (or `'product'`) whenever images are queried or created through the association.
- **`GET /images/:id`** вЂ” fetches the image first, then conditionally looks up `User` or `Product` by `imageableId` and exposes it under the `imageable` key.
