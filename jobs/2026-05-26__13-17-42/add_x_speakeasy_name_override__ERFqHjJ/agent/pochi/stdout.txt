When Speakeasy generates the SDK, that operationId becomes the method name on the generated client (e.g. `client.users.pst_user_create(...)`), which is confusing for SDK consumers. Speakeasy provides the OpenAPI extension `x-speakeasy-name-override` so SDK authors can rename the generated method without rewriting the underlying `operationId` in the source spec. Your job is to edit the seeded `openapi.yaml` so that the method generated from the `POST /users` operation will be named `createUser` instead of `pst_user_create`. ## Requirements * Edit `/home/user/project/openapi.yaml` in place to add an `x-speakeasy-name-override` rule that renames the `pst_user_create` operation to `createUser`. * Either form documented by Speakeasy is acceptable: * Operation-level: `paths./users.post.x-speakeasy-name-override: createUser`. * Global-level: a top-level `x-speakeasy-name-override` list containing an entry that matches operationId `pst_user_create` (an exact-match regex is fine) with `methodNameOverride: createUser`. * Do **not** change the existing `operationId` `pst_user_create` itself. The renaming must come from the extension. * Do **not** remove or rename any existing operations, paths, or schemas. Keep the OpenAPI version as `3.0.x` and keep the `User` schema component intact. * After your edits, the spec must remain valid: running `speakeasy lint openapi -s /home/user/project/openapi.yaml --non-interactive` must still exit with code 0. ## Implementation Hints * The Speakeasy CLI is preinstalled and available on the `PATH` as `speakeasy`. `yq` is also available for YAML inspection. * Refer to the Speakeasy documentation for `x-speakeasy-name-override` on the "Customize methods" page; both global and operation-level placements are documented and accepted. * When using the global form, remember that the matching key is `operationId` (a regex) and the value key is `methodNameOverride`. * When using the operation-level form, the extension is a plain string value placed directly on the operation object. * You only need to modify `openapi.yaml`.
