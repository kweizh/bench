## What the script does

1. **OTel SDK initialization** — `NodeSDK` is started first (before any tracing imports) with `LangfuseSpanProcessor` as the span exporter. The processor reads `LANGFUSE_PUBLIC_KEY`, `LANGFUSE_SECRET_KEY`, and `LANGFUSE_BASE_URL` from the environment automatically.

2. **Root observation** — `startActiveObservation` opens a span named `weather-assistant-${ZEALT_RUN_ID}` with the user message as `input` and the simulated assistant reply as `output`.

3. **Nested generation** — Inside the root span, `rootSpan.startObservation("llm-call", { input, model: "gpt-4o-mini" }, { asType: "generation" })` creates a child `GENERATION` observation. It is updated with `output` and `model` before `.end()` is called.

4. **Trace ID capture** — `getActiveTraceId()` is called while inside the active context, printed to stdout, and written to `/home/user/myproject/output.log` as `Trace ID: <id>`.

5. **Reliable flush** — `await sdk.shutdown()` is called after the trace completes, ensuring all spans are exported before the process exits.
