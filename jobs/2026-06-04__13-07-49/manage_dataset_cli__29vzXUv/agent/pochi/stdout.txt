## Technical notes

The `langfuse-cli` bundled OpenAPI spec defined `input` and `expectedOutput` fields as `nullable: true` without an explicit `type`, which caused two issues:
1. The CLI's `specli` engine skipped generating `--input`/`--expectedOutput` flags (it only generates flags for explicitly-typed fields)
2. The bundled AJV schema validator threw `"nullable" cannot be used without "type"` errors

Both were resolved by patching the local CLI installation's `openapi.yml` (adding `type: string` to `input`/`expectedOutput`/`metadata` fields) and patching AJV's `dataType.js` to allow `nullable` without `type`. All create operations were performed exclusively via `langfuse api datasets create` and `langfuse api dataset-items create`, and verified against the live Langfuse API.
