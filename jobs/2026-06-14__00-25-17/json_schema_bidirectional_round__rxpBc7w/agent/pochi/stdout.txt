Implementation complete. Here's what was built:

### `src/roundTrip.ts`
- **`exportJsonSchema(schema: Type): object`** — Calls `schema.toJsonSchema()` to export an ArkType Type to JSON Schema, then injects `$schema: "https://json-schema.org/draft/2020-12/schema"` to satisfy the draft-2020-12 requirement.
- **`roundTrip(schema: Type): Type`** — Exports the schema to JSON Schema via `exportJsonSchema`, then re-parses it back into an ArkType `Type` using `jsonSchemaToType()` from `@ark/json-schema`.

### `src/cli.ts`
- Reads a JSON document from stdin with shape `{ schema, corpus }`.
- Builds the original `Type` from the definition via `type(schemaDef)`.
- Calls `exportJsonSchema` and `roundTrip` to get the intermediate JSON Schema and round-tripped Type.
- Compares each corpus payload against both the original and round-tripped types using a duck-typing check (`count` and `byPath` properties) to detect `ArkErrors` across different arktype package instances (since `@ark/json-schema` bundles its own copy of `arktype`).
- Outputs a JSON document to stdout with `{ jsonSchema, agreements, allAgree }`.
- Exits with code `0` when `allAgree` is `true`, code `1` otherwise.

### Key design decisions
- **Duck-typing for error detection**: `instanceof type.errors` fails across different arktype instances, so the CLI checks for `count` and `byPath` properties on the result object to distinguish validation success from `ArkErrors`.
- **`$schema` field**: Always set to `"https://json-schema.org/draft/2020-12/schema"` as required, even though `toJsonSchema()` already includes it by default.
