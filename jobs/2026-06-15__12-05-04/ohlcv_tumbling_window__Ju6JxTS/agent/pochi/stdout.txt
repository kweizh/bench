## Architecture

The dataflow has six stages:

```
CSVSource  →  key_on(symbol)  →  fold_window(EventClock + TumblingWindower)
                                       │
                          window_out.down (OHLCV values)
                          window_out.meta (WindowMetadata with open_time)
                                       │
                          label + merge into one keyed stream
                                       │
                          stateful_flat_map (joiner — buffer until both arrive)
                                       │
                          FileSink → output.jsonl
```

### Key design decisions

| Concern | Approach |
|---|---|
| **Event time** | `EventClock` driven by the `timestamp` column; naive timestamps are treated as UTC |
| **Windowing** | `TumblingWindower(length=1 min, align_to=Unix epoch)` — windows align exactly to wall-clock minutes |
| **OHLCV accumulation** | `fold_window` with a custom `OHLCV` dataclass; `builder` creates an empty bar, `folder` updates it per trade, `merger` combines partials |
| **Metadata join** | `fold_window` emits OHLCV values on `window_out.down` and `WindowMetadata` (carrying `open_time`) on `window_out.meta`; these are merged into one keyed stream and buffered via `stateful_flat_map` until both arrive, then serialised to JSON |
| **Late data** | `wait_for_system_duration=10s` — windows stay open for 10 s of wall-clock time after the last event, giving out-of-order events a chance to arrive |
