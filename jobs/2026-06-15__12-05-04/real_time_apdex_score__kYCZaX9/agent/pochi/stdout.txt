### Key design decisions
- **`EventClock`** uses the `timestamp` field from each event to drive watermark advancement, making the windows purely event-time based.
- **`TumblingWindower`** is aligned to the Unix epoch (`1970-01-01T00:00:00Z`) so window boundaries are always clean multiples of 10 seconds regardless of input timestamps.
- **`ApdexAccumulator`** is a dataclass tracking `satisfied / tolerating / frustrated` counts; `score` is computed lazily as a property.
- Window open time is derived as `ALIGN_TO + window_id * WINDOW_LENGTH` (the integer `window_id` emitted by `fold_window.down` is the window's index from the alignment point).
- The output file is truncated at startup so repeated runs produce a fresh file.
