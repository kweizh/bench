## Pipeline Architecture

```
FileSource (JSONL)
    │
    ▼
filter_map "parse"
    Parse JSON lines → (player_id, score)
    │
    ▼
key_on "key_by_player"
    Key stream by player_id
    │
    ▼
stateful_map "player_max_score"
    Per-player state: track all-time maximum score
    Emits (player_id, current_max) on every update
    │
    ▼
map "to_global_key"
    Re-route all items to key "__global__"
    so a single worker sees the complete picture
    │
    ▼
fold_final "topk_fold"
    Min-heap of size K (O(log K) per insert)
    Deduplicates players on every update
    Emits once at EOF
    │
    ▼
map "serialise"
    Sort heap descending → JSON string
    │
    ▼
MemoryCollectorSink → write to output file
```

## Key design decisions

| Concern | Solution |
|---|---|
| Max score per player | `stateful_map` with `state = max(state, score)` — O(1) per event |
| Global Top-K | `fold_final` with a **min-heap of size K** — O(log K) per update, O(K) memory |
| Duplicate players in heap | On each update, stale entry for that player is removed before re-inserting |
| Single-worker aggregation | All items are re-keyed to `"__global__"` before `fold_final` |
| Empty / edge inputs | `filter_map` drops blank lines; empty stream → `[]` output |
