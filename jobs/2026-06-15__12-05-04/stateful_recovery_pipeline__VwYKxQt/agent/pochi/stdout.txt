## How the pipeline works

```
FileSource(input.jsonl)
  → op.map      parse_json          str → dict
  → op.key_on   key_by_sensor       dict → (sensor_id, dict)    [key = string ✓]
  → win.fold_window  fold_temps     EventClock + SlidingWindower(60s/30s, align 2026-01-01Z)
      builder  = list()             ← pure Python, picklable ✓
      folder   = acc.append(r["temp"])
      merger   = acc1 + acc2
  ┌── .down  (sensor_id, (win_id, [temps]))
  └── .meta  (sensor_id, (win_id, WindowMetadata))
  → op.map  rekey_{down,meta}   compound key  "S1|42"
  → op.join join_meta           (key, ([temps], WindowMetadata))
  → op.map  compute_stats       mean + population stddev
  → FileSink(output.jsonl)
```

## Key design decisions for picklability

* **Accumulator** is a plain `list` — no lambdas, no closures, no unpicklable types.
* `_builder`, `_folder`, `_merger` are module-level named functions (not lambdas) so they pickle as references.
* `_get_ts` (the `EventClock` ts_getter) is also a named module-level function.
* All state structures stored in the SQLite recovery DB are pure Python scalars confirmed via `pickle.loads`.
