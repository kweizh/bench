## Architecture

```
FileSource (input.jsonl)
    │
    ▼  op.filter_map("parse_and_key")
    │  JSON parse → (user_id, event_dict)   ← keyed stream required by stateful_map
    │
    ▼  op.stateful_map("fraud_state_machine")
    │  Per-user state machine  →  (user_id, "FRAUD_ALERT" | SENTINEL)
    │
    ▼  op.filter("filter_alerts")
    │  Drop sentinel values
    │
    ▼  op.map("format_output")
    │  (user_id, event_dict) → (user_id, JSON string)
    │
    ▼  FileSink (output.jsonl)
```

## State machine (`fraud_state_machine`)

| Current state | Event | Condition | Next state | Output |
|---|---|---|---|---|
| any | `logout` | — | `LOGGED_OUT` | — |
| any | `login` | — | `LOGGED_IN`, reset counter | — |
| `LOGGED_OUT` | `transaction` | — | `LOGGED_OUT` (ignored) | — |
| `LOGGED_IN`/`SUSPICIOUS` | `transaction` | `ts - login_ts > 300s` | `LOGGED_OUT` (session expired) | — |
| `LOGGED_IN`/`SUSPICIOUS` | `transaction` | `amount < 1000` | unchanged | — |
| `LOGGED_IN`/`SUSPICIOUS` | `transaction` | `amount ≥ 1000`, counter < 2 | `SUSPICIOUS`, counter++ | — |
| `LOGGED_IN`/`SUSPICIOUS` | `transaction` | `amount ≥ 1000`, counter == 2 | `LOGGED_OUT`, reset | **`FRAUD_ALERT`** |

Key design choices:
- **Effectively-immutable state**: every transition returns a new `UserState` dataclass instance, never mutating in place — satisfying Bytewax's snapshot requirement.
- **`op.stateful_map`**: Bytewax stores the returned state object between events for the same key. `None` is passed for the first event; the mapper initialises a fresh `UserState`.
- **Sentinel pattern**: `stateful_map` must emit one output per input; non-alert events return a sentinel string that is dropped by a downstream `op.filter`.
- **`FileSink` keyed tuples**: `FileSink` requires `(key, value)` tuples from upstream; the value is what gets written. The format step wraps the JSON string back into a tuple.
