**Pipeline stages in `enrichment.py`:**

1. **Static dataset load** — `products.json` is opened once at module import time into the global `PRODUCTS` dict (available on every worker before any items flow).

2. **Input** — `FileSource("transactions.jsonl")` feeds raw lines into the dataflow.

3. **Parse + enrich** (`filter_map` → `parse_and_enrich`) — each line is parsed from JSON, looked up in `PRODUCTS`, and transformed into `(category, (timestamp, revenue))`. Lines whose `product_id` is absent in the catalogue return `None` and are dropped (t4/p3 in the test data).

4. **Windowed aggregation** (`fold_window`) — an `EventClock` driven by the per-item timestamp with `wait_for_system_duration=timedelta(0)` (suitable for a bounded file source) and a `TumblingWindower` of 1-minute length aligned to `2026-01-01T00:00:00Z` (an exact hour boundary). Revenue is accumulated with a simple float addition fold.

5. **Metadata join** — `fold_window` returns a `WindowOut`; its `.down` stream carries `(category, (window_id, revenue))` and its `.meta` stream carries `(category, (window_id, WindowMetadata))`. Both are re-keyed on `"category:window_id"` and joined with `op.join` so the `open_time` (window start) is available for formatting.

6. **Format + output** — each joined record is serialised to the required JSON shape and written to stdout via `StdOutSink`.
