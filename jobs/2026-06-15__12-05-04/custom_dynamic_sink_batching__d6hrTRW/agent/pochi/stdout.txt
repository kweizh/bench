### Key classes

**`RotatingFileSinkPartition(StatelessSinkPartition)`**  
Per-worker partition that manages file rotation:
- Tracks `_part_number` (starts at 0) and `_records_in_current_file`.
- `_open_next_part()` closes the current file handle and opens a fresh one named `output-<run-id>-worker-<worker_index>-part-<N>.jsonl`.
- `write_batch()` iterates items **one-by-one**. Before writing each item it checks `_records_in_current_file >= 20`; if so, it increments the part counter and rotates to a new file. This correctly handles batches that span multiple file boundaries.
- `close()` flushes and closes the last open file on dataflow completion.

**`RotatingFileSink(DynamicSink)`**  
Instantiates one `RotatingFileSinkPartition` per worker via `build(step_id, worker_index, worker_count)`, so every worker writes to its own isolated file set.
