## `/home/user/graph_pipeline/pipeline.py`

**Architecture:**

```
FileSource (input_edges.jsonl)
  → map "parse_json"       — each line → {"u":…, "v":…} dict
  → key_on "single_partition" — all edges keyed to "global"
  → stateful_map "union_find"  — UnionFind state per key, emits JSON string
  → output "write_events"      — FileSink writes (key, json) tuples
```

**Key design decisions:**

1. **Single partition routing** — `key_on` assigns every edge the constant key `"global"`, so `stateful_map` maintains one `UnionFind` instance covering the entire graph (required for correctness).

2. **UnionFind** — pure-Python, path-compressed, union-by-size; only `dict` fields → fully picklable for Bytewax SQLite recovery snapshots.

3. **`FileSink` tuple contract** — `FixedPartitionedSink` expects `(key, value)` 2-tuples; `stateful_map` already outputs `("global", json_string)`, so no extra key-stripping step is needed.

4. **Output filename** — resolved at module load from `ZEALT_RUN_ID` env var (defaults to `"default"`).

**Execution (SQLite recovery enabled):**
```bash
mkdir -p recovery_db
python -m bytewax.recovery recovery_db 1
```
