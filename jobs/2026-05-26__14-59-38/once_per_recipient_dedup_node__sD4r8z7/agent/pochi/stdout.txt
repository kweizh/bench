When set to `once_per_recipient`, the second (and any subsequent) trigger of the same workflow for the same recipient is dropped, no matter how many times your backend calls the trigger API. This is the recommended primitive for one-shot lifecycle notifications such as a welcome email. You will build a small Node.js script that uses the Knock **Management API** (`@knocklabs/mgmt`) to create and activate a one-step email workflow with `once_per_recipient` deduplication enabled, and then uses the Knock **API SDK** (`@knocklabs/node`) to trigger the workflow twice for the same recipient. The Knock backend should only deliver a single email even though the script triggers the workflow twice.
