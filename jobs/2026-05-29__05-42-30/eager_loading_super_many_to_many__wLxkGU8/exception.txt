# Super Many-to-Many Eager Loading

## Background
In Sequelize, a "Super Many-to-Many" relationship involves two primary models connected through a junction model, where the junction model itself has its own associations (e.g., it belongs to a third model). Eager loading data from the junction model's associations requires setting up the relationships correctly and using nested `include` statements.

## Requirements
- Create a Node.js script that sets up a SQLite database with four models: `Student`, `Course`, `Enrollment` (the junction model), and `Semester`.
- `Student` and `Course` have a Many-to-Many relationship through `Enrollment`.
- `Enrollment` has a `belongsTo` relationship with `Semester` (i.e., a student enrolls in a course for a specific semester).
- Insert the following test data:
  - Student: name "Alice"
  - Courses: title "Math 101", title "History 101"
  - Semesters: name "Fall 2023", name "Spring 2024"
  - Alice is enrolled in "Math 101" for "Fall 2023", and "History 101" for "Spring 2024".
- Write a query to fetch the Student "Alice", including her Courses, and for each Course, include the Semester information from the `Enrollment` junction model.
- Output the fetched Student data as a JSON string to stdout.

## Implementation Hints
- Define the `Enrollment` model explicitly and use it in the `belongsToMany` calls for both `Student` and `Course`.
- To make eager loading on the junction model work, you must define the One-to-Many associations between `Student` and `Enrollment`, and `Course` and `Enrollment` in addition to the Many-to-Many associations (this is the standard "Super Many-to-Many" setup in Sequelize).
- Define `Enrollment.belongsTo(Semester)`.
- When querying `Student.findOne()`, use nested `include` to fetch `Course`, and include the `Enrollment` model to fetch the `Semester`.
