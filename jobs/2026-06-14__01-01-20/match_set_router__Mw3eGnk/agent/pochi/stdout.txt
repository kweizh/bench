Two modules were created in `/home/user/myproject`:

### `router.ts` — ArkType `match`-based event router
- Uses ArkType 2.2.0's `match` API with a combination of object-literal keys (for `string`, `number`, `string[]`) and fluent `.case()` calls (for discriminated nested objects).
- Six cases plus `default: "assert"`:
  1. **`string`** → `text:<length>`
  2. **`number`** → `num:<value>`
  3. **`string[]`** → `list:<length>`
  4. **Click on button** (`kind: "click"`, `target.type: "button"`, `target.id: string`) → `btn:<id>`
  5. **Click on link** (`kind: "click"`, `target.type: "link"`, `target.href: string.url`) → `link:<href>`
  6. **Submit** (`kind: "submit"`, `payload.formId: string`, `payload.valid: boolean`) → `submit:<formId>:<valid>`
- Non-matching events cause `route()` to throw a `TraversalError` via the `"assert"` default.
- The `kind` field is used as a discriminator with distinct literal values (`"click"` vs `"submit"`), and within click events, `target.type` further discriminates between `"button"` and `"link"`.

### `cli.ts` — stdin/stdout CLI
- Reads a single JSON document `{"events": [...]}` from stdin.
- Iterates events in order, applies `route()` to each.
- Prints routed strings on stdout, one per line.
- On the first `TraversalError`, prints `ERR <message>` and stops processing.
- Always exits with code 0.
- `tsconfig.json` was updated with `allowImportingTsExtensions: true` to support `.ts` extension imports.
