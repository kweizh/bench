## What was built

### `pipeline.py` — dataflow named `flow`

**Custom source (`PlainCSVSource` / `_CSVPartition`)**  
A `FixedPartitionedSource` / `StatefulSourcePartition` pair that reads a headerless CSV file line-by-line. The partition snapshot stores the line count, making it resume-safe.

**Stateful logic (`moving_average`)**  
```python
def moving_average(state, temperature):
    if state is None:
        state = ()
    new_state = (state + (temperature,))[-3:]   # immutable new tuple, never mutated
    return new_state, sum(new_state) / len(new_state)
```
Returns a brand-new tuple each call (no in-place mutation), which is required for Bytewax's recovery snapshots.

**Dataflow pipeline steps**
1. `op.input` — emit raw CSV lines from `input.csv`
2. `op.map` — parse each line into `(sensor_id, temperature)`
3. `op.key_on` — key stream by `sensor_id` (string key, as required)
4. `op.map_value` — strip the key from the value tuple, leaving just the float
5. `op.stateful_map` — apply `moving_average` per sensor key
6. `op.map_value` + `op.map` — format as `"sensor_id,X.XX"` keyed string
7. `op.output` — write to `output.csv` via `FileSink`
