## What was built

### `pipeline.py`
A Bytewax 0.21.1 dataflow with these stages:

| Stage | Details |
|---|---|
| **Input** | `JsonlSource` – `FixedPartitionedSource` reading `input.jsonl` line-by-line; resume state is a plain `int` (line offset) |
| **Parse** | `json.loads` maps each raw line to a `dict` |
| **Key** | `op.key_on` keys by `sensor_id` (string) |
| **Clock** | `EventClock(ts_getter=_get_event_time, wait_for_system_duration=0s)` — parses `time` field to UTC-aware `datetime` |
| **Window** | `SlidingWindower(length=60 s, offset=30 s, align_to=2026-01-01T00:00:00Z)` |
| **Fold** | `win.fold_window` with `_builder / _folder / _merger` — accumulates temperatures into a plain `list[float]` |
| **Join** | `op.join` on a composite `"sensor_id\|win_id"` key to merge the accumulator stream with the `WindowMetadata` stream |
| **Format** | Computes mean + population stddev, serialises to JSON |
| **Output** | `JsonlSink` – `DynamicSink` appending to `output.jsonl` |

### Picklability
All callable state (builder/folder/merger/ts_getter) are **module-level named functions** — no lambdas. The accumulator is a plain Python `list[float]`. Every object serialises cleanly with `pickle`, as confirmed by the 14 recovery snapshot rows written to `recovery_dir/part-0.sqlite3`.
