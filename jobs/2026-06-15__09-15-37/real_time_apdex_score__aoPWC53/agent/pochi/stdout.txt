## Dataflow pipeline

```
FileSource("input.jsonl")
  → map("parse")           # JSON line → (service, record) + parsed _ts
  → fold_window(           # Tumbling 10 s event-time window per service key
        EventClock(ts_getter),
        TumblingWindower(10s, align_to=epoch),
        builder=build_accumulator,   # {satisfied, tolerating, frustrated}
        folder=fold_apdex,           # classify each response time
        merger=merge_accumulators
    )
  → map("rekey_down/meta") # compound key "service|window_id" for join
  → join("join_meta")      # pair accumulator with WindowMetadata (open_time)
  → map("format")          # compute Apdex, serialise JSON → (key, str)
  → FileSink("output.jsonl")
```

## Key design decisions

- **`EventClock`** drives time from the `timestamp` field in each record, not wall-clock time, making the flow correct for batch and out-of-order data.
- **`TumblingWindower`** is aligned to the Unix epoch so window boundaries (`00:00`, `00:10`, `00:20`, …) are deterministic and reproducible across runs.
- **`fold_window`** emits two parallel streams (`down` = result, `meta` = `WindowMetadata` with `open_time`). These are re-keyed to `"service|window_id"` and joined so the window start timestamp is available at output time.
- **`FileSink`** requires `(key, value)` pairs; `format_output` returns `(compound_key, json_string)` to satisfy this.
