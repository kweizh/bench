## Key Design Decisions
- The `reached` setter resets `_has_destination` when set to `false`, so the test harness's `_reset_agent` (which sets `reached = false`) properly clears state before the second navigation attempt
- `target_reached` is guarded by `_has_destination` to prevent premature firing when `NavigationAgent2D.target_position` equals the agent's current position during reset
