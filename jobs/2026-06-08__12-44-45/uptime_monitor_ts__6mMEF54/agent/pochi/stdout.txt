## Implemented Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST /site` | Add a site | Inserts site into DB, returns `{id, url, is_up}` |
| `GET /site` | List sites | Returns `{sites: [{id, url, is_up}, ...]}` |
| `POST /check` | Trigger checks | Publishes a message per site to Pub/Sub, returns `{status: "ok"}` |

## How it works

1. **`POST /check`** fetches all site IDs from PostgreSQL and publishes a `CheckMessage{site_id}` to the `site-check` Pub/Sub topic.
2. The **subscriber** processes each message: fetches the site URL from the DB, performs an HTTP GET with a 10-second timeout, and updates `is_up` (true if status 200–299, false otherwise).
