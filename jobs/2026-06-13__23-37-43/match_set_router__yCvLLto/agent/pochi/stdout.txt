## Files created

### `src/router.ts`
Contains the `route` matcher built with ArkType 2.2.0's `match` API:

- **Cases 1–3** (`string`, `number`, `string[]`) use the object-literal shorthand — these primitive/array type strings are valid embeddable record keys in `match({...})`.
- **Cases 4–6** (the three object-shape cases) use the fluent `.case(type({...}), handler)` API because object shapes cannot be embedded as record-key strings. Each object case references a distinct `kind` literal (`'click'` / `'submit'`) so the discriminator is observable in the source.
- **Case 5** (link click) uses `"string.url"` to require a syntactically valid URL for `target.href`.
- **`.default("assert")`** finalises the chain — ArkType throws a `TraversalError` for any input that matches none of the six cases.

### `cli.ts`
Reads a single JSON document `{"events": [...]}` from stdin, iterates the events in order, calls `route(event)` for each, and:
- Prints each result on its own stdout line.
- On the first `TraversalError` (or any error), prints `ERR <message>` and stops.
- Always exits with code `0`.
