## Architecture

### `src/pipeline.ts` — three composed ArkType morphs

**Stage 1 – `csvToRawRows`** (`string => raw-row[]`)  
A `type("string", "=>", fn)` morph that performs all structural parsing:
- Rejects empty input or a missing/malformed header (case-sensitive exact match against `id,age,email,signupAt`)
- Rejects rows with ≠ 4 cells
- Returns an array of `{ id, age, email, signupAt }` where every value is still a `string`

**Stage 2 – `userRowType`** (object schema with inline morphs)  
- `id` → `"string.uuid"` — built-in UUID constraint  
- `age` → `type("string.integer.parse").pipe(type("0 <= number.integer <= 150"))` — parses the digit string to a number, then range-validates it; two piped ArkType types  
- `email` → `"string.email"` — built-in email constraint  
- `signupAt` → `"string.date.iso.parse"` — validates ISO-8601 format **and** morphs the string into a real `Date` instance entirely within ArkType

**Stage 3 – `csvToRawRows.pipe(userRowArrayType)`**  
Chains the structural morph into the array validator so the full pipeline is a single `Type<(In: string) => To<UserRecord[]>>`.

### `cli.ts`
Reads stdin verbatim (no trimming), passes the raw string to `csvPipeline`, and:
- On success: prints `VALID` then a JSON array (where `Date` instances serialise via `toISOString()`)
- On failure: prints `INVALID: <summary>`
- Always exits with code `0`
