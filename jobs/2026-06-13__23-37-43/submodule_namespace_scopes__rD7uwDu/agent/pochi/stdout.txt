## `/home/user/myproject/src/schema.ts`

A single `scope({...})` call defines all four submodule aliases using ArkType's dot-namespaced key feature:

```ts
const myScope = scope({
  "db.User":               { id: "string.uuid", name: "1<=string<=50",  orgId: "string.uuid" },
  "db.Org":                { id: "string.uuid", name: "1<=string<=100" },
  "api.CreateUserRequest": { user: "db.User",   token: "32<=string<=256" },
  "api.CreateOrgRequest":  { org:  "db.Org",    adminUserId: "string.uuid" },
});
```

Key points:
- **Single `scope({...})` call** — no separate scopes for `db` or `api`.
- **String-alias cross-references** — `"user: "db.User"` and `"org: "db.Org"` reference by name, never inlining the shapes.
- **ArkType built-in keywords** — `string.uuid` for UUID validation, `1<=string<=50` etc. for length constraints.
- After exporting the scope, thin `Module` wrappers are assembled so consumers get **true nested dot-access** (`schemaModule.db.User`, `schemaModule.api.CreateUserRequest`).
