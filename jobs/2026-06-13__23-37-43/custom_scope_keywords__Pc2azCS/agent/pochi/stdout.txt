### `src/keywords.ts`

Defines all three custom keywords and the `Order` schema inside a **single `scope({...})` call**, then exports the resulting module members:

- **`creditCard`** — `type("string").matching(/^\d{13,19}$/)` for the digit/length constraint, then **`.narrow(...)`** for the Luhn checksum (the mandatory narrow predicate lives here).
- **`usPhone`** — `type("string").matching(/^\+?1?[\s-]?\(?\d{3}\)?[\s-]?\d{3}[\s-]?\d{4}$/)` using the exact spec regex.
- **`slug`** — `type("string").matching(/^[a-z0-9-]{3,64}$/)` for charset + length, then **`.narrow(...)`** to reject a leading or trailing dash.
- **`Order`** — an object schema that references all three keywords by bare alias name (`"slug"`, `"usPhone"`, `"creditCard"`, `"number > 0"`).

### `cli.ts`

Reads one JSON document from stdin, calls `Order.assert(parsed)`:
- **Valid** → prints `VALID` then the JSON-stringified object on the next line.
- **Invalid** (schema error or JSON parse error) → prints a single line beginning with `INVALID: `.
- Always exits with code `0`.
