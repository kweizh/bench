The event-emitter module is implemented at `/home/user/myproject` with two files:

### `src/emit.ts`

Defines the validated `emit` function using ArkType 2.2's `type.fn` machinery (`InternalTypedFn` — the exact object every `type.fn(...)( impl )` call produces internally).

The parameter schema is expressed as a **union of two concrete tuple shapes** using `type(...)`:

| Shape | Tuple |
|---|---|
| Without payload | `[eventName, timestamp, ...tags]` |
| With payload | `[eventName, timestamp, payload, ...tags]` |

Each constraint:
- **eventName**: `"string.alphanumeric & 1 <= string <= 50"` — alphanumeric, 1–50 chars
- **timestamp**: `"number.integer >= 0"` — non-negative integer  
- **payload**: `{ kind: "string", data: "unknown" }` — object shape
- **tags**: `"(1 <= string <= 30)[]"` — variadic array of 1–30-char strings

> **Why a union instead of `[T?, ...R[]]`?** ArkType 2.2 validates the optional slot positionally: a string passed at position [2] is tested against the optional `{kind,data}` schema and fails. The union of two concrete shapes correctly accepts both call patterns.

### `cli.ts`

Reads a single JSON line `{"args":[...]}` from stdin, spreads `args` into `emit(...)`, and prints:
- `OK <json>` on success  
- `ERR <message>` on `TraversalError` (invalid input)

Both cases exit with code 0.
