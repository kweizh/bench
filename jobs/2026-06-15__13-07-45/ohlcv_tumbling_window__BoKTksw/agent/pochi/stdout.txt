**Architecture:**

The dataflow pipeline consists of 7 steps:
1. **FileSource** — reads the CSV file line-by-line
2. **filter** — skips the header row (`timestamp,symbol,price,volume`)
3. **map** — parses each CSV line into a `Trade` dataclass (timestamp, symbol, price, volume)
4. **key_on** — keys the stream by ticker symbol for grouping
5. **fold_window** — applies a 1-minute `TumblingWindower` with `EventClock` to accumulate `OHLCVState` per symbol per window
6. **filter_map** — transforms `(symbol, (window_id, OHLCVState))` tuples into JSON-serializable dicts with `window_start`, `symbol`, `open`, `high`, `low`, `close`, `volume`
7. **output** — writes each dict as a JSON line via a custom `JSONLSink`

**Key design decisions:**
- `EventClock` with `wait_for_system_duration=timedelta(seconds=0)` ensures windows close immediately after the last event is processed (no waiting for late data in batch mode)
- `TumblingWindower` aligned to epoch (`1970-01-01T00:00:00Z`) so all minute boundaries are exact
- `OHLCVState` tracks `first_timestamp` to derive `window_start` by truncating to the minute, and `last_timestamp` for proper merge ordering
- The `merger` function handles the case where `fold_window` might need to merge accumulators (though tumbling windows don't typically merge, this ensures correctness if the runtime ever requires it)

**Usage:** `python run.py --input input.csv --output output.jsonl`
