## Architecture

The dataflow pipeline:

1. **FileSource** → reads JSONlines file line-by-line
2. **map (parse_json)** → deserializes JSON strings to dicts
3. **key_on** → keys the stream by `user_id` for partitioned state
4. **stateful_map** → runs the fraud detection state machine per user
5. **filter_map_value** → filters out `None` results (no alert), keeps only fraud alerts
6. **map_value** → serializes alert dicts back to JSON strings
7. **FileSink** → writes alerts to output JSONlines file

## State Machine (`UserState`)

The state is an immutable `@dataclass(frozen=True)` with three fields:
- `state`: `LOGGED_OUT` | `LOGGED_IN` | `SUSPICIOUS`
- `login_timestamp`: when the user last logged in
- `large_txn_count`: number of transactions >= 1000 since login

**Transitions:**
- `LOGGED_OUT` + `login` → `LOGGED_IN` (records timestamp, resets count)
- `LOGGED_IN`/`SUSPICIOUS` + `transaction` with amount >= 1000 → increments count, transitions to `SUSPICIOUS`
- Count reaches 3 within 300s of login → emits `FRAUD_ALERT`, resets to `LOGGED_OUT`
- Any `logout` → `LOGGED_OUT`
- Transaction > 300s after login → resets to `LOGGED_OUT` (transaction ignored)
- Transactions while `LOGGED_OUT` are silently ignored
