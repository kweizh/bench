# Context Arithmetic (Intersection) with Alchemyst AI Python SDK ## Background Alchemyst AI is a Context Engine that lets agents organize ingested documents into composable groups via the `group_name` metadata array. At search time, supplying multiple group values acts as a set intersection (AND) — only documents that belong to *all* of the specified groups survive the filter. This is the foundation of "Context Arithmetic". In this task you will build a small Python CLI that ingests several documents tagged with overlapping `group_name` values, then performs a filtered search and prints the matching documents as JSON. The CLI must demonstrate the intersection semantics correctly using the official `alchemystai` Python SDK. IMPORTANT references the implementor should consult: * Context Arithmetic: https://getalchemystai.com/docs/advanced/context-arithmetic.md * Python API Reference (v0.10.0): https://raw.githubusercontent.com/Alchemyst-ai/alchemyst-sdk-python/refs/tags/v0.10.0/api.md * Quickstart (Metadata / group\_name section): https://getalchemystai.com/docs/getting-started/quickstart.md Note: In the Python SDK, both add and search use `group_name` (snake\_case) consistently — unlike the TypeScript SDK which uses `groupName` for search. ## Requirements * Build a Python CLI at `/home/user/myproject/main.py` that uses the `alchemystai` Python SDK. * The CLI must accept one repeated/variadic argument `--groups` that takes one or more group names (e.g. `--groups eng v1`). * On every invocation, the CLI must first ingest **at least 3** documents whose `group_name` arrays overlap such that the intersection semantics can be verified. Use these exact documents (content and group assignments): * Document A: belongs to groups `["eng", "v1"]` * Document B: belongs to groups `["eng", "v2"]` * Document C: belongs to groups `["docs", "v1"]` * After ingestion, the CLI must call `client.v1.context.search(...)` with `metadata={"group_name": [<groups from --groups>]}` and print to stdout a single JSON array containing one object per matching document. Each object must contain at minimum the field `file_name` (string) corresponding to the document's `metadata.file_name`. Any other fields are allowed but ignored by the verifier. * The CLI must read the API key from the environment variable `ALCHEMYST_AI_API_KEY`.
