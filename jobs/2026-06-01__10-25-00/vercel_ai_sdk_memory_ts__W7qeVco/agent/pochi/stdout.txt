# Cross-Session Memory CLI with Vercel AI SDK + Alchemyst (TypeScript) ## Background You will build a small Node.js TypeScript CLI that uses the Vercel AI SDK (`ai` package) wrapped with Alchemyst's `withAlchemyst` middleware from `@alchemystai/aisdk` to give the LLM persistent, cross-session memory. Alchemyst is a Context Engine that gives LLMs durable memory keyed by `(userId, sessionId)`. When `withAlchemyst(...)` wraps `generateText`, every call to the wrapped function automatically (a) retrieves past memories for the `userId`, (b) injects them into the prompt, and (c) persists the new turn. This lets the agent remember a preference established in one session and reuse it in a completely different session for the same user. The CLI must demonstrate this with two phases: - `--phase establish`: the user tells the assistant a personal dietary preference. The assistant acknowledges and the memory is written. - `--phase recall`: a *new session id* is used, the user asks an open question, and the assistant must answer using the preference that was stored in the previous phase. ## Requirements - A Node.js project (npm) at `/home/user/vercel-ai-memory` written in TypeScript. - Production dependencies that **must** be declared in `package.json`: - `ai` (Vercel AI SDK) - `@ai-sdk/openai` (Vercel AI SDK OpenAI provider) - `@alchemystai/aisdk` (Alchemyst middleware) - TypeScript build that emits the runnable entrypoint to `dist/main.js`. - The CLI accepts a single argument `--phase <establish|recall>` and reads: - `ALCHEMYST_AI_API_KEY` (Alchemyst API key) - `OPENAI_API_KEY` (OpenAI API key) - `ZEALT_RUN_ID` (run id used to namespace the user and session ids; see Parallel-Run Safety below) - Use the wrapped function `withAlchemyst(generateText, { apiKey: process.env.ALCHEMYST_AI_API_KEY })` and call it with an OpenAI model from `@ai-sdk/openai` (for example `openai('gpt-4o-mini')` or another small OpenAI chat model). **Do not** call OpenAI directly without the Alchemyst wrapper. - Every call to the wrapped `generateText` MUST pass both `userId` and `sessionId`.
