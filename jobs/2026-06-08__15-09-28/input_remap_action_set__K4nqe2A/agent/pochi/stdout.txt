## Notable design decisions

- **`InputRemapper.gd`** captures deep-copied default events in `_ready()` so `reset_to_defaults()` is always consistent regardless of how many rebinds have happened.
- `rebind_action` erases **only** `InputEventKey` and `InputEventJoypadButton` events before adding the new one, leaving any other event types (e.g. joypad axes) untouched.
- `save_to_file` / `load_from_file` use `ConfigFile`; key and joypad-button events are serialised by `keycode` / `button_index` integers, which survive process restarts without relying on resource serialisation.
- **`RemapButton.gd`** guards `get_viewport().set_input_as_handled()` against a null viewport (safe for headless/test contexts) and uses `is_connected` guards so `_ready()` is idempotent.
- `project.godot` uses the `run/main_scene` key (not `config/run/main_scene`) — the correct Godot 4 project-settings path for `application/run/main_scene`.
