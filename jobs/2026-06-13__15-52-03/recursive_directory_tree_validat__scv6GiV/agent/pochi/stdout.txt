### `src/validator.ts`

Defines the schema with `scope(...).export()` and exports `validateDirectoryTree`:

```
fileNode      — { "+": "reject", name: "string > 0", size: "number.integer > 0" }
directoryNode — { name: "string > 0", "size?": "never", "children?": "(fileNode | directoryNode)[]" }
treeNode      — "fileNode | directoryNode"
```

Key design decisions:
- `"+": "reject"` on `fileNode` — rejects any extra properties (including `children`), enforcing the file-has-no-children rule.
- `"size?": "never"` on `directoryNode` — ensures a node carrying both `size` and `children` is rejected from the directory branch as well, so the union rejects it entirely.
- Children are typed as `(fileNode | directoryNode)[]` (explicit union string) to work around a runtime issue in arktype 2.2.0 with self-referential alias names in array position.
