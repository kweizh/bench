The `index.js` file now initializes an Express app with a middleware that:
1. Creates a new PocketBase instance pointing to `http://127.0.0.1:8090`.
2. Extracts the `pb_auth` cookie from the incoming request's `cookie` header and loads it into the `pb.authStore`.
3. If the auth store is valid, it attempts to refresh the authentication state using `pb.collection('users').authRefresh()`. If the refresh fails (e.g., due to an expired or invalid token), it catches the error and clears the auth store.
4. Serializes the updated auth state back to a cookie and attaches it to the response via the `Set-Cookie` header.
5. Exposes the `pb` instance via `res.locals.pb` for route handlers.

The `/profile` route uses this middleware and returns the user's `id` and `email` if the auth state is valid, or a `401 Unauthorized` error if the token is invalid or missing. The `Set-Cookie` header properly reflects the current authentication state (either refreshed or cleared) in both cases.
