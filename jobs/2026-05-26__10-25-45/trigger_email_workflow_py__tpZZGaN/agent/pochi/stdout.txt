The script is idempotent — re-running it for the same `ZEALT_RUN_ID` succeeds (upserts overwrite, user identify merges, new trigger returns a new run ID). The key issue discovered during development was that the Management API requires:
- The workflow payload to be wrapped under a `"workflow"` key
- `template.settings` to be present (non-null) for email steps
- An explicit `PUT /v1/commits` call to commit the changes before the workflow can be triggered via the regular API
