# Personalize Knock workflow emails with per-recipient trigger data (Python) ## Background Your team uses https://knock.app for transactional notifications. Knock lets a single workflow trigger call notify multiple recipients, and each recipient can receive its own merged data via the inline-recipient `$trigger_data` field. You need to ship a Python script that defines (upserts) a Knock email workflow whose template renders per-recipient data, and then triggers it once for multiple inline recipients so each person receives an email customized with their role and dashboard URL. The Knock account already has an email channel with key `mailtrap` configured for delivery. Inbound mail is captured through Gmail using sub-addressed inboxes.
