# Trigger a Knock Email Workflow with the Python SDK ## Background Knock is a notification infrastructure platform. Your team wants a small Python automation that programmatically creates a Knock workflow that sends a single email notification (using the pre-configured `mailtrap` channel), and then triggers it for a specific user so the user receives a welcome email at a Gmail inbox the QA team monitors. The Knock account already has two channels configured for this environment: * An in-app feed channel with key `in-app` * An email channel with key `mailtrap` You will: 1. Use the Knock Management API (authenticated with `KNOCK_SERVICE_TOKEN`) to upsert a workflow in the `development` environment. 2. Activate the workflow so it can be triggered. 3. Use the Knock Python SDK (`knockapi`) to identify a user and trigger the workflow for that user. 4. Persist the resulting `workflow_run_id` and recipient details to a log file. ## Requirements * Implement the automation as a single Python script located at `/home/user/knock_task/run.py`. * Read the following environment variables in the script: * `ZEALT_RUN_ID` — the run id used to scope all created resources. * `KNOCK_SERVICE_TOKEN` — Management API token (account-level). * `KNOCK_API_TOKEN` — environment-level secret API key for triggering workflows. * `MAILTRAP_DOMAIN` — sender domain accepted by Mailtrap. * `GMAIL_USER_NAME` — Gmail account used to verify inbound email. * Create (upsert) a Knock workflow in the `development` environment whose key is `welcome-email-${ZEALT_RUN_ID}` and whose name is `Welcome Email ${ZEALT_RUN_ID}`. The workflow must have exactly one step: an email channel step that uses the channel key `mailtrap`. * The email step `ref` must be `welcome_email`. * The email subject template must be exactly: `Welcome to Knock, {{ recipient.name }}!`. * The email HTML body template must render the recipient's name and a dashboard URL provided through trigger `data` (`{{ data.dashboard_url }}`). * The email sender address must be `sender-${ZEALT_RUN_ID}@${MAILTRAP_DOMAIN}` (configured via the email step's `settings.from_email_address`). * The workflow `trigger_data_json_schema` must require a string `dashboard_url` field. * Activate the workflow in `development` so it is runnable. * Identify a user in Knock with: * `id`: `user-${ZEALT_RUN_ID}` * `name`: `Knock User ${ZEALT_RUN_ID}` * `email`: `${GMAIL_USER_NAME}+receiver-${ZEALT_RUN_ID}@gmail.com` * Trigger the workflow once for that recipient with `data.dashboard_url` set to `https://app.example.com/dashboard/${ZEALT_RUN_ID}`. * Write a log file at `/home/user/knock_task/output.log` containing the following two lines (in this order): ``` Workflow key: welcome-email-<run-id> Workflow run id: <workflow_run_id> Recipient id: user-<run-id> Recipient email: <gmail_user_name>+receiver-<run-id>@gmail.com ``` ## Implementation Hints * Use HTTP `PUT https://control.knock.app/v1/workflows/{key}?environment=development` (with `Authorization: Bearer $KNOCK_SERVICE_TOKEN`) or the management endpoint via `requests` / `httpx` to upsert the workflow. You can also use any officially supported management SDK if installed. * After upserting, send a request to activate the workflow in the `development` environment so it transitions to `active`. * Install the `knockapi` package and use `Knock(api_key=...)`. Identify the user via `client.users.update(user_id=..., ...)` (or equivalent identify call) and trigger the workflow via `client.workflows.trigger(key=..., recipients=[...], data=...)`. * The workflow run identifier returned by the trigger call must be stored verbatim in the log file. * Make the script idempotent: re-running it for the same `ZEALT_RUN_ID` must continue to succeed (upserts and identifies must overwrite existing data). ## Acceptance Criteria * Project path: /home/user/knock\_task * Ensure the script is executed and the resulting artifacts exist. * Log file: /home/user/knock\_task/output.log * The log file must contain a line in the format `Workflow key: welcome-email-${ZEALT_RUN_ID}`. * The log file must contain a line in the format `Workflow run id: <uuid>` where `<uuid>` is the value returned by the Knock trigger API. * The log file must contain a line in the format `Recipient id: user-${ZEALT_RUN_ID}`. * The log file must contain a line in the format `Recipient email: ${GMAIL_USER_NAME}+receiver-${ZEALT_RUN_ID}@gmail.com`. * The workflow `welcome-email-${ZEALT_RUN_ID}` must exist in Knock's `development` environment with `active = true` and contain exactly one channel step whose `channel_key` is `mailtrap` and whose `ref` is `welcome_email`. * An email matching the configured subject must arrive in the Gmail inbox of `${GMAIL_USER_NAME}+receiver-${ZEALT_RUN_ID}@gmail.com`.
