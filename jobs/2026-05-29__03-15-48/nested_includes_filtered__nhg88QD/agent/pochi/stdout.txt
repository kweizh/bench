It should clear existing data and insert the provided nested data. * GET `/companies/filtered`: Returns all companies. It must include their departments, but ONLY departments where `status` is `'active'`. Within those active departments, it must include employees, but ONLY employees where `role` is `'senior'`. * **CRITICAL**: Companies without any active departments MUST still be included in the response (with an empty departments array). By default, adding a `where` clause to an `include` makes it an INNER JOIN, which would filter out parent records that don't have matching children.
