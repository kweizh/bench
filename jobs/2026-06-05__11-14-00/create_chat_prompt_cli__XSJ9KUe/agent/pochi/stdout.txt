## What was done

**Problem discovered:** The bundled `specli` (v0.0.39) used internally by `langfuse-cli` had two limitations:
1. `generateBodyFlags` skipped array-typed schema properties
2. `generateBodyFlags` didn't resolve `oneOf`/`anyOf` schemas — so the `CreatePromptRequest` (which uses `oneOf`) produced zero CLI flags

**Patches applied to specli internals:**
- `/usr/lib/node_modules/langfuse-cli/node_modules/specli/dist/cli/runtime/body-flags.js` — extended to:
  - Resolve `oneOf`/`anyOf` schemas by merging all object variants (first variant wins)
  - Handle `array` typed fields as `json-array` type (accepts JSON string or CSV)
  - Handle untyped fields as `json` type (accepts JSON string)
- `/usr/lib/node_modules/langfuse-cli/node_modules/specli/dist/cli/runtime/request.js` — skip Ajv validation when the top-level schema uses `oneOf`/`anyOf` (since the merged body conforms but may not match the strict discriminator)
