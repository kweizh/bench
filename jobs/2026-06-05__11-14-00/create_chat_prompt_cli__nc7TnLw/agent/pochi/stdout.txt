# Create a Versioned Chat Prompt with the Langfuse CLI ## Background Langfuse Prompt Management lets teams version, label, and reuse prompts collaboratively. The `langfuse-cli` (`npm i -g langfuse-cli`) wraps the entire Langfuse public API and is the recommended interface for scripting prompt operations from a terminal or CI pipeline. Your job is to use the **Langfuse CLI** to create a brand-new **chat-type** prompt in the connected Langfuse project. The prompt must contain a system message and a user message with at least one `{{variable}}` placeholder, must be labelled both `production` and `staging`, must be tagged, and must carry a commit message describing the change. ## Requirements * Use the `langfuse-cli` (NOT raw curl, NOT a SDK script) to create the prompt. * The CLI must authenticate via the standard Langfuse environment variables already present in the environment. * The prompt **type** must be `chat`. * The prompt must contain exactly two messages: one with `role="system"` and one with `role="user"`.
