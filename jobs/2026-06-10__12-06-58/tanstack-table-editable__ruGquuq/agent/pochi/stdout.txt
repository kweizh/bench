## How It Works

| Behaviour | Implementation |
|---|---|
| Table rendering | `useReactTable` + `getCoreRowModel` with 5 columns (ID, Name, Email, Role, Actions) |
| Edit mode toggle | `editingId` state; matching row renders `<EditableRow>` instead of normal cells |
| Form management | `useForm` from `@tanstack/react-form` with `defaultValues` pre-filled from the row |
| Validation | `onChange` validators on every field – Name (required, ≥2 chars), Email (required + regex), Role (required) |
| Error display | Inline `⚠ message` shown below each field after it is touched |
| Save | `form.handleSubmit()` → `onSave(updatedUser)` → replaces row in state, exits edit mode |
| Cancel | `onCancel()` → resets `editingId` to `null`; form is discarded since the component unmounts |
| Only one edit at a time | All other "Edit" buttons are `disabled` while `editingId !== null` |
