Events arrive as a JSON document on stdin and the router must dispatch each one to the appropriate handler based on its set-theoretic shape, including primitives, arrays, and discriminated nested objects. ## Requirements * Implement a matcher named `route` constructed via `match({...})` (from `arktype`) that handles AT LEAST the following six explicit cases plus a default: 1. A bare `string` → produces `text:<length>` (where `<length>` is the string's character length). 2. A bare `number` → produces `num:<value>`. 3. A `string[]` → produces `list:<length>`. 4. Lines printed for previously routed events MUST already be on stdout. * The CLI MUST always exit with status code 0, regardless of whether a default rejection occurred. ## Implementation Hints * Consult the ArkType docs for `match` and decide how to express each case (some are convenient as object-literal keys in the case record, while nested-object cases may be easier to express via the fluent `.case({...}, handler)` API).
