## What was implemented

### `main.go`

**Hook registration** — `app.OnRecordCreateRequest("posts").BindFunc(...)` intercepts every API `POST /api/collections/posts/records` request before the record is saved.

**Validation** — if `title` is empty/missing, the hook returns `router.NewBadRequestError("Title cannot be empty", nil)`, which causes PocketBase to respond with HTTP 400.

**Slug generation** — the custom `slugify` function:
1. Decomposes unicode characters via NFD and strips non-spacing combining marks (accents/diacritics) using `golang.org/x/text/transform` + `unicode.Mn`.
2. Lowercases the result.
3. Replaces runs of non-alphanumeric characters with a single hyphen.
4. Trims leading/trailing hyphens.

Example: `"Hello, World! 2026"` → `"hello-world-2026"`, `"Héllo Wörld"` → `"hello-world"`.

**Chain propagation** — `e.Next()` is returned to continue the middleware/hook chain so the record is actually persisted to the database.

### API behaviour

| Request | Response |
|---|---|
| `POST /api/collections/posts/records` with `{"title": "My First Post"}` | `200 OK` — record includes `"slug": "my-first-post"` |
| `POST /api/collections/posts/records` with `{"title": ""}` or no title | `400 Bad Request` — `"Title cannot be empty"` |

> **Note:** The `posts` collection (with `title` and `slug` text fields) must be created in the PocketBase admin UI (or via migration) before the hook can operate. The hook itself only sets field values — collection schema management is handled separately.
