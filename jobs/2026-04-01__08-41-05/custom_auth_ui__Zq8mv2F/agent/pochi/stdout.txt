# Wasp Custom Auth UI ## Background While Wasp provides high-level Auth UI components, you can also build your own UI and call the auth actions manually. In this task, you will implement a completely custom login and signup interface. ## Requirements - **Authentication**: Enable `usernameAndPassword` authentication. - **Custom UI**: - Do **not** use Wasp's `LoginForm` or `SignupForm` components. - Create custom React forms in `src/SignupPage.tsx` and `src/LoginPage.tsx`. - Use Wasp's `signup` and `login` functions from `wasp/client/auth` to handle form submissions. - **Pages**: - `SignupPage`: A form with `username` and `password` inputs and a "Sign Up" button. - `LoginPage`: A form with `username` and `password` inputs and a "Log In" button. - `MainPage` (protected): Displays "You are logged in as [username]" and a "Logout" button. ## Implementation Guide 1. **Initialize Project**: - Create a new Wasp project in `/home/user/custom-auth-ui` using the minimal template. 2. **Configure App**: - Set up `auth` in `main.wasp`. - Define `route` and `page` for `MainPage`, `LoginPage`, and `SignupPage`. 3. **Implement Custom UI**: - In `SignupPage.tsx`, use React state for inputs and call `signup({ username, password })` followed by `login({ username, password })` on submit. - In `LoginPage.tsx`, call `login({ username, password })` on submit. 4. **Database Migration**: - Run `wasp db migrate-dev`. ## Constraints - **Project Path**: `/home/user/custom-auth-ui` - **Start Command**: `wasp start` - **Port**: 3000
