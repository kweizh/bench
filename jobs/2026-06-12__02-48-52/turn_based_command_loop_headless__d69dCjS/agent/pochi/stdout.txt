### Key Implementations:
1. **`ScriptedInput` (Custom `MockInput`)**:
   - Extends `com.badlogic.gdx.backends.headless.mock.input.MockInput`.
   - Manages the queue of commands parsed from the input file.
   - Exposes a `tick()` method to advance the command queue on each render loop, mapping valid commands (`N`, `S`, `E`, `W`, `PICK`, `QUIT`) to their respective `Input.Keys` and invalid/unknown commands to `-1`.
2. **`GameListener` (`ApplicationListener`)**:
   - Swaps `Gdx.input` with the `ScriptedInput` instance in `create()`, satisfying the requirement to replace it before the first render tick.
   - Parses the map format flexibly, ignoring blank lines and lines starting with `#` while extracting all tokens.
   - Contains the core game logic inside `render()`. It reads the current command mapped key, updates the player position/inventory (enforcing map bounds and item pickups), and appends the formatted turn result to the transcript builder.
   - Quits the application cleanly with `Gdx.app.exit()` when either `QUIT` is encountered or the command queue is exhausted. It then writes the entire transcript with `Gdx.files.absolute(...).writeString(...)` and signals the main thread to terminate via a `CountDownLatch`.
3. **`HeadlessLauncher`**:
   - Parses the CLI arguments (`--map`, `--commands`, `--transcript`).
   - Configures `HeadlessApplicationConfiguration` with `updatesPerSecond = 1000` to run the loop as fast as possible.
   - Starts the `HeadlessApplication` and waits for the game loop to finish using the latch, ensuring the JVM exits cleanly with code `0` only after the transcript is written.
