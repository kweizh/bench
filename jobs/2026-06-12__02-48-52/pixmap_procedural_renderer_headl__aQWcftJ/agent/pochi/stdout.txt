Combined with the `gdx-backend-headless` back-end, these primitives can run inside a CI job, a server process, or a unit test — no OpenGL context is required as long as the code never touches `Gdx.gl*`. Your job is to build a small, reproducible command-line tool that boots a `HeadlessApplication`, draws a procedural image into a `Pixmap` based on a text command script, and writes the result to a PNG file using `PixmapIO.writePNG`. ## Requirements * Implement a Gradle-based Java project that depends on `com.badlogicgames.gdx:gdx:1.14.2`, `com.badlogicgames.gdx:gdx-backend-headless:1.14.2`, and the desktop natives classifier `com.badlogicgames.gdx:gdx-platform:1.14.2:natives-desktop`. * Provide an `ApplicationListener` (e.g., extending `ApplicationAdapter`) that is started by a `HeadlessApplication` and performs all rendering and file I/O work inside the libGDX life-cycle (not from `main` directly). * Read a plain-text command script (referred to as the *input file*) and execute its drawing commands against a single `Pixmap`. ## How it works

**`PixmapRenderer.java`**  
- Validates CLI args (`<input-script> <output.png>`).  
- Creates a `HeadlessApplicationConfiguration` with `updatesPerSecond = -1` (no render loop).  
- Boots `new HeadlessApplication(listener, config)`.  
- Waits on a `CountDownLatch` that `DrawingListener` releases when `create()` finishes.  
- Prints the single summary line **after** the latch (guaranteeing the PNG is flushed), then `System.exit()`.

**`DrawingListener.java`**  
- All work runs inside `create()` on the libGDX thread — safe for `Pixmap` / `PixmapIO`.  
- Reads the script line-by-line; skips blanks and `#`-comments.  
- Dispatches to a `switch` on the uppercased command token:
  - `SIZE w h` — allocates a `Pixmap(w, h, RGBA8888)` (must be first non-comment line).
  - `FILL r g b a` — `pixmap.setColor(…); pixmap.fill()`.
  - `RECT x y w h r g b a` — `pixmap.fillRectangle(…)`.
  - `LINE x1 y1 x2 y2 r g b a` — `pixmap.drawLine(…)`.
  - `CIRCLE cx cy radius r g b a` — `pixmap.fillCircle(…)`.
  - `PIXEL x y r g b a` — `pixmap.drawPixel(…)`.
  - Integer 0–255 color components are divided by 255 before `setColor` (which takes `[0,1]` floats).
- Writes the result with `PixmapIO.writePNG(Gdx.files.absolute(outputPath), pixmap)`.
- `dispose()`s the pixmap, releases the latch, and calls `Gdx.app.exit()`.

**`run.sh`**  
```bash
bash run.sh <input-script> <output.png>
```
- Converts both paths to absolute (so `Gdx.files.absolute()` works from any CWD).  
- `cd`s to the project root, then calls:
  ```
  gradle --no-daemon --offline --quiet run --args="<abs-input> <abs-output>"
  ```
