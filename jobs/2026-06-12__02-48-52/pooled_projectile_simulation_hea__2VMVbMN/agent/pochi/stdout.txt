## Implementation highlights

- **Backend**: `HeadlessApplication` with `updatesPerSecond = 0` (max throughput). LWJGL3 is never on the classpath.
- **Pool**: Anonymous `Pool<Projectile>` subclass. `pool.obtain()` on spawn, `pool.free(p)` on grounding, `pool.getFree()` for the final summary.
- **Physics**: Exactly one tick per `render()` call (fixed ╬Ф = 1). Per tick: spawn тЖТ apply gravity to velocity тЖТ add velocity to position тЖТ ground-check (`y <= floor_y`).
- **Output ordering**: `activeProjectiles` is kept sorted by spawn-id via `insertSortedById()`, so both the active-state lines and the grounded lines are naturally in ascending id order.
- **Peak tracking**: Measured immediately after spawning (before grounding) so simultaneous maximums are captured correctly.
- **Clean exit**: `CountDownLatch` in `dispose()` (after writer flush/close) releases `main()` from `latch.await()`, preventing early JVM exit.
- **Scenario I/O**: File read via `Gdx.files.absolute(path)`; all output lines end with `\n` only.
