The framework's math package (`com.badlogic.gdx.math`) is fully usable from this backend and ships ready-made 2D collision primitives: `Rectangle.overlaps(Rectangle)`, `Circle.overlaps(Circle)`, and `Intersector.overlaps(Circle, Rectangle)`. Servers and offline batch jobs frequently lean on these primitives to detect collisions, validate level layouts, or pre-compute spatial indices without ever opening a window. ## Key design decisions

| Concern | Approach |
|---|---|
| Thread synchronisation | `CountDownLatch(1)` counted down in `create()`'s `finally` block after `Gdx.app.exit()`; main thread calls `done.await()` |
| Error signalling | `volatile String errorMessage` on the listener; Launcher prints it to stderr and calls `System.exit(1)` |
| Overlap detection | Exclusively `Rectangle.overlaps(Rectangle)`, `Circle.overlaps(Circle)`, `Intersector.overlaps(Circle, Rectangle)` |
| Output ordering | Within each pair: `compareTo` ≤ 0 → (a,b) else (b,a); pairs list sorted by primary then secondary id |
| Headless config | `updatesPerSecond = -1` (non-blocking loop) |

## Verified behaviours

- **Happy path**: pairs emitted tab-separated, lexicographically ordered, followed by `total_overlaps=N`, file ends with `\n`, exit 0.
- **No overlaps**: file contains only `total_overlaps=0\n`, exit 0.
- **Duplicate id**: `Error: duplicate id <id>` → stderr, exit 1, output file not written.
- **Unknown shape keyword / wrong token count**: `Error: invalid shape line: <raw line>` → stderr, exit 1.
- **Non-parseable number**: same invalid-line error, exit 1.
- **Non-positive width/height/radius (including zero)**: same invalid-line error, exit 1.
