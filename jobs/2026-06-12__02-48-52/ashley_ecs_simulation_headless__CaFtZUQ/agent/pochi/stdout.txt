## Project Structure

- **`build.gradle`** — Gradle build with all required dependencies (gdx 1.13.1, gdx-backend-headless 1.13.1, gdx-platform natives-desktop, ashley 1.7.4) and the `application` plugin for `gradlew run`

- **`src/main/java/sim/VelocityComponent.java`** — Ashley Component with `x` and `y` float fields
- **`src/main/java/sim/MovementSystem.java`** — Ashley `IteratingSystem` that applies `position += velocity * deltaTime` for all entities matching `Family.all(Position, Velocity)`
- **`src/main/java/sim/SimulationListener.java`** — `ApplicationListener` that loads the scenario, creates entities, ticks the engine with fixed `dt = 1/60`, and prints final state after `TICKS` render calls
- **`src/main/java/sim/Main.java`** — Entry point that creates a `HeadlessApplication` (with `updatesPerSecond = 60`), passes the scenario path, and joins the headless thread before exiting
- **`run.sh`** — Shell entry point that runs `gradlew -q run` with stderr silenced, ensuring only simulation output appears on stdout

## Key Design Decisions

- **Deterministic time step**: Uses a fixed `1.0f / 60.0f` delta per tick, never reads `Gdx.graphics.getDeltaTime()`
- **Thread safety**: A `done` flag prevents double-printing when `Gdx.app.exit()` (asynchronous) allows one extra `render()` call
- **Entity ordering**: Entities are stored in input order so output matches scenario file order
- **Locale.ROOT formatting**: All floats use `String.format(Locale.ROOT, "%.3f", ...)` to avoid locale-specific decimal separators
