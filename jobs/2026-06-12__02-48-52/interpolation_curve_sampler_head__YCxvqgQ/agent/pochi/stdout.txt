# Interpolation Curve Sampler with libGDX Headless Backend ## Background libGDX ships a tweening utility, `com.badlogic.gdx.math.Interpolation`, that exposes named easing curves (`linear`, `smooth`, `smoother`, and many more) for animating values between two endpoints. The class is part of `com.badlogicgames.gdx:gdx` and does not require an OpenGL context, so it can be driven by the headless backend (`com.badlogicgames.gdx:gdx-backend-headless`) and used inside CI / server pipelines. The program must reject any other curve name with a non-zero exit code and an error message on stderr. * Make the program runnable through Gradle's `application` plugin so it can be launched with `./gradlew run --args="<config-path> <output-path>"`. ## Implementation Hints * Look at `HeadlessApplication` and `HeadlessApplicationConfiguration`. ## Key design decisions

| Concern | Decision |
|---|---|
| Headless pacing | `config.updatesPerSecond = 0` — loop runs as fast as possible with no wall-clock sleep |
| Thread join | `CountDownLatch(1)` decremented inside `dispose()`; `main()` calls `latch.await()` |
| Exact endpoints | `t=0.0f` at `i=0`, `t=1.0f` at `i=samples-1` (forced, no float drift) |
| Curve validation | Pre-checked in `main()` via a plain `switch` before the `HeadlessApplication` is created; exits with code 1 + stderr message, output file never touched |
| Locale | `Locale.ROOT` in all `printf` calls → decimal separator is always `.` |
| Line endings | `%n` output is run through `.replace("\r\n", "\n")` before `Files.write()` → LF-only on all platforms |
| Gdx.files exercise | `create()` opens the config via `Gdx.files.absolute(configPath).read()` |
