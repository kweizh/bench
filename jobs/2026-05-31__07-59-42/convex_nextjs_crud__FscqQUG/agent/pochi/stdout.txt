## Project Structure

- **`convex/schema.ts`** — Defines the `tasks` table with `text` (string), `isCompleted` (boolean), and `runId` (string) fields, plus an index on `runId`
- **`convex/tasks.ts`** — Contains four Convex functions:
  - `getTasksByRunId` — query that filters tasks by `runId`
  - `createTask` — mutation to add a task with `text` and `runId`
  - `toggleTask` — mutation to toggle `isCompleted` status
  - `deleteTask` — mutation to delete a task
- **`src/app/ConvexClientProvider.tsx`** — Client component wrapping the app with `ConvexProvider`
- **`src/app/layout.tsx`** — Root layout with `ConvexClientProvider`
- **`src/app/page.tsx`** — Main UI with task input, add button, task list with toggle/delete

## Environment Variables

- **`NEXT_PUBLIC_ZEALT_RUN_ID`** — Read from `ZEALT_RUN_ID` (defaults to `"default"`), used to filter and create tasks
- **`NEXT_PUBLIC_CONVEX_URL`** — Set automatically by `npx convex dev` in `.env.local`; can also be overridden from `CONVEX_URL`

## data-testid Attributes

- `task-input` — Text input for new task text
- `add-button` — Button to add a new task
- `task-item` — Each task item in the list
- `toggle-button` — Button/checkbox to toggle task completion
- `delete-button` — Button to delete a task
