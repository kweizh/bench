# Run a Langfuse Experiment with Item-Level Evaluators (TypeScript) ## Background https://langfuse.com/ is an open-source AI engineering platform that provides observability, prompt management, and evaluation for LLM applications. The TypeScript SDK's `experiment.run` API loops a task function over a dataset, automatically traces every execution, and lets you attach **item-level evaluators** whose results are emitted as scores on each trace in Langfuse. In this task you will build a small Node.js (TypeScript) project that runs a Langfuse experiment over a **local** dataset using the modular JS/TS SDK packages, with a custom task function and two custom evaluators, and then record the resulting trace IDs so they can be inspected via the Langfuse API. ## Requirements * Create a TypeScript Node.js project at `/home/user/myproject` that uses the modular Langfuse JS/TS SDK packages (`@langfuse/client`, `@langfuse/otel`, `@opentelemetry/sdk-node`). * Initialize OpenTelemetry with `LangfuseSpanProcessor` before running the experiment so traces are exported to Langfuse. * Define an in-process **local** dataset of exactly 3 items, each with an `input` string and an `expectedOutput` string. * Define a task function that returns the uppercase form of the item's `input` string. The task function must **not** call any external LLM provider. * Define **two** item-level evaluators that are passed to `experiment.run` via the `evaluators` option: * One named `accuracy` that returns numeric value `1` when the task output exactly equals `expectedOutput` (case-sensitive) and `0` otherwise. * One named `response_length` that returns the number of characters of the task `output`. * Run the experiment via `langfuse.experiment.run({...})`.
