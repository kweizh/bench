# Cross-Session Memory CLI with Vercel AI SDK + Alchemyst (TypeScript) ## Background You will build a small Node.js TypeScript CLI that uses the Vercel AI SDK (`ai` package) wrapped with Alchemyst's `withAlchemyst` middleware from `@alchemystai/aisdk` to give the LLM persistent, cross-session memory. Alchemyst is a Context Engine that gives LLMs durable memory keyed by `(userId, sessionId)`. When `withAlchemyst(...)` wraps `generateText`, every call to the wrapped function automatically (a) retrieves past memories for the `userId`, (b) injects them into the prompt, and (c) persists the new turn. This lets the agent remember a preference established in one session and reuse it in a completely different session for the same user. The CLI must demonstrate this with two phases: - `--phase establish`: the user tells the assistant a personal dietary preference. The assistant acknowledges and the memory is written. - `--phase recall`: a *new session id* is used, the user asks an open question, and the assistant must answer using the preference that was stored in the previous phase. ## Requirements - A Node.js project (npm) at `/home/user/vercel-ai-memory` written in TypeScript. - Production dependencies that **must** be declared in `package.json`: - `ai` (Vercel AI SDK) - `@ai-sdk/openai` (Vercel AI SDK OpenAI provider) - `@alchemystai/aisdk` (Alchemyst middleware) - TypeScript build that emits the runnable entrypoint to `dist/main.js`. - The CLI accepts a single argument `--phase <establish|recall>` and reads: - `ALCHEMYST_AI_API_KEY` (Alchemyst API key) - `OPENAI_API_KEY` (OpenAI API key) - `ZEALT_RUN_ID` (run id used to namespace the user and session ids; see Parallel-Run Safety below) - Use the wrapped function `withAlchemyst(generateText, { apiKey: process.env.ALCHEMYST_AI_API_KEY })` and call it with an OpenAI model from `@ai-sdk/openai` (for example `openai('gpt-4o-mini')` or another small OpenAI chat model). **Do not** call OpenAI directly without the Alchemyst wrapper. - Every call to the wrapped `generateText` MUST pass both `userId` and `sessionId`. ## Key Implementation Details

1. **Alchemyst wrapper**: Uses `withAlchemyst(generateText, { apiKey: process.env.ALCHEMYST_AI_API_KEY })` to wrap the Vercel AI SDK's `generateText` function, enabling persistent cross-session memory.

2. **Parallel-run safety**: User and session IDs are namespaced with `ZEALT_RUN_ID`:
   - `userId = vercel-memory-user-${ZEALT_RUN_ID}`
   - `sessionId = establish-${ZEALT_RUN_ID}` (establish phase)
   - `sessionId = recall-${ZEALT_RUN_ID}` (recall phase — different session)

3. **Phase prompts**:
   - `--phase establish`: Sends "Please remember this about me: I am vegan and I am allergic to peanuts. Acknowledge that you will remember."

4. **Error handling**: Fails fast with non-zero exit and stderr message if `ALCHEMYST_AI_API_KEY`, `OPENAI_API_KEY`, or `ZEALT_RUN_ID` are missing, or if `--phase` is invalid.

5. **Build**: Uses esbuild for fast bundling (the `@alchemystai/aisdk` package ships raw TypeScript, making `tsc` impractically slow). Run `npm run build` to rebuild.
