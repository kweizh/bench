Created `/home/user/myproject/main.py` — a Python CLI that implements shared team memory with the Alchemyst AI SDK (v0.10.0).

### What it does

1. **Validates environment variables** — exits with non-zero code if `ALCHEMYST_AI_API_KEY` or `ZEALT_RUN_ID` is missing.
2. **Derives identifiers** from `ZEALT_RUN_ID`:
   - Session: `session-${ZEALT_RUN_ID}`
   - Alice: `alice-${ZEALT_RUN_ID}`
   - Bob: `bob-${ZEALT_RUN_ID}`
3. **Seeds two memory entries** into the shared session via `client.v1.context.memory.add()`:
   - Alice's phrase: *"Alice prefers Python for data processing pipelines"*
   - Bob's phrase: *"Bob recommends PostgreSQL with TimescaleDB for time-series storage"*
   - Seeding is idempotent (re-running adds duplicate entries, which doesn't break retrieval).
4. **Retrieves shared context** via `client.v1.context.search()` (the correct v0.10.0 method — **not** `memory.search` which doesn't exist in this SDK version). Uses the `--user-id` and `--query` arguments, with `body_metadata` scoped to the session and `metadata="true"` to include content. Retries up to 10 times with 3-second delays to allow asynchronous indexing.
5. **Prints results** in the required format:
   ```
   USER: <user-id>
   SESSION: session-<run-id>
   RETRIEVED:
   - <snippet 1>
   - <snippet 2>
   ```
6. **Exits 0** on success, **non-zero** on failure (missing env vars or both phrases not found).

### Usage

```bash
export ALCHEMYST_AI_API_KEY=<your-key>
export ZEALT_RUN_ID=<run-id>

python3 main.py --user-id alice-${ZEALT_RUN_ID} --query "technology preferences"
python3 main.py --user-id bob-${ZEALT_RUN_ID} --query "database recommendations"
```

### Key design decisions

- **`client.v1.context.search`** is used instead of `client.v1.context.memory.search` (which doesn't exist in `alchemystai==0.10.0`), per the authoritative API reference at the v0.10.0 tag.
- **`memory.add`** uses `contents=[{"content": ...}]` (list of content objects) and `session_id` — matching the actual v0.10.0 SDK signature.
- **Retry logic** handles the asynchronous indexing delay mentioned in the requirements.
- The venv is at `/home/user/myproject/.venv` with `alchemystai==0.10.0` installed.
