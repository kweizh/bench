## How It Works

1. **CLI parsing**: Uses `minimist` to read `--topic <string>` from `process.argv`
2. **Environment variables**: Reads `ALCHEMYST_AI_API_KEY`, `OPENAI_API_KEY`, and `ZEALT_RUN_ID`
3. **Seed articles**: Calls `v1.context.add` for each of the 5 research articles, with `metadata.fileName` namespaced by `ZEALT_RUN_ID` (e.g., `b2b_article_ai_agents_<run-id>.md`). Uses `context_type: 'resource'` and `scope: 'internal'`. Catches 409 Conflict errors for idempotent reruns.
4. **Context search**: Calls `v1.context.search` with the provided topic to retrieve relevant snippets
5. **Newsletter generation**: Calls OpenAI Chat Completions API (`gpt-4o-mini`) with a system prompt requiring Markdown output with at least 3 section headings, grounded in the retrieved context
6. **Output**: Writes the generated Markdown to `output/newsletter.md` (creates directory if needed)
