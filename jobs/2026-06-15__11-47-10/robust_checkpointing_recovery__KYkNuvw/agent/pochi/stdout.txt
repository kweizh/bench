   - To support sequential runs with different input/output files without triggering Bytewax's default `assert path == str(self._path)` constraint, I implemented custom `CustomFileSource` (inheriting from `FixedPartitionedSource`) and `CustomFileSink` (inheriting from `FixedPartitionedSink`).
   - These custom implementations use a tuple of `(file_path, offset)` for the resume state. On resume, if the file path is different from the previous execution, the source and sink gracefully start reading/writing from the beginning of the new file, while maintaining the state of the stateful operators.
   - The dataflow parses the CSV lines, applies the stateful `running_max` operator (using `op.stateful_map`), and formats the output back to `key,running_max` strings.
