### Implementation Details

1. **Bytewax 0.21.1 Framework Usage**:
   - The dataflow is built using Bytewax's modern operator API (`bytewax.operators` as `op`).
   - We use `FileSource` to read the input JSONlines file line-by-line and `FileSink` to write the output JSONlines file.
   - We parse the input lines using `json.loads` and key the stream by `user_id` using `op.key_on` before passing it to the stateful operator.
   - Stateful processing is implemented using `op.stateful_map` to track each user's state independently.
   - We filter out empty results (where no fraud alert was generated) using `op.filter_value`, and format the alerts back into JSON strings using `op.map_value` before passing them to the output sink.

2. **State Machine Logic**:
   - **Immutability**: Following Bytewax guidelines, the state is represented as a dictionary and effectively copied into a new dictionary during each transition to ensure immutability.
   - **Transitions**:
     - Users start in `LOGGED_OUT`.
     - A `login` event transitions the user to `LOGGED_IN` and records the `login_timestamp`.
     - A `transaction` event with `amount >= 1000` while `LOGGED_IN` or `SUSPICIOUS` increments `large_transaction_count` and transitions the state to `SUSPICIOUS`.
     - Accumulating 3 such transactions within 300 seconds of the last `login` event emits a `FRAUD_ALERT` and resets the state to `LOGGED_OUT`.
     - Any `logout` event immediately resets the state to `LOGGED_OUT`.
     - If a transaction occurs more than 300 seconds after the login, the state resets to `LOGGED_OUT` and the current transaction is ignored.

3. **Verification and Testing**:
