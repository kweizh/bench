## Architecture

```
Main (Node)
тФЬтФАтФА PeerRoot_0  тЖР server SceneMultiplayer attached here
тФВ   тФЬтФАтФА Players
тФВ   тФФтФАтФА Spawner (MultiplayerSpawner, spawn_path="../Players")
тФЬтФАтФА PeerRoot_1  тЖР client 0 SceneMultiplayer attached here
тФВ   тФЬтФАтФА Players
тФВ   тФФтФАтФА Spawner
тФФтФАтФА PeerRoot_N  тЖР client N-1 SceneMultiplayer
    тФЬтФАтФА Players
    тФФтФАтФА Spawner
```

Each peer gets its own `SceneMultiplayer` + `ENetMultiplayerPeer` instance, attached to a distinct subtree via `SceneTree.set_multiplayer(api, path)`. All peers run in the same headless process on loopback.

## Key design decisions

- **Authority via `_enter_tree`**: `player.gd` reads its own `name` (set to `peer_id` before `add_child`) and calls `set_multiplayer_authority()` in `_enter_tree`. This avoids the Godot 4.3 "no network ID" error that occurs when authority is changed in `_ready`.
- **Auto-spawn mode**: The server instantiates `player.tscn` and calls `Players.add_child(player)`. The `MultiplayerSpawner` detects children whose scene path is in its spawnable list and broadcasts the spawn packet automatically тАФ no `spawn_function` required.
- **Position authority**: Each client sets `player.position = Vector2(my_id, my_id)` on its own authoritative copy once the spawned node appears; the `MultiplayerSynchronizer` (replication_mode=Always) replicates this to all peers.
- **Score RPC**: `@rpc("any_peer", "call_local")` `update_score(value: int)` is called once per client after its player is observed, routed through the correct `SceneMultiplayer` context automatically.
