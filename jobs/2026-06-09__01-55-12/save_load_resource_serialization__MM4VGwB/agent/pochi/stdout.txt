### `scripts/SaveManager.gd`
```
class_name SaveManager
extends RefCounted
```

**`save_to_disk(data, path, binary) -> int`**  
Strips any existing `.tres`/`.res` extension from `path` via `_stem()`, re-attaches the correct one (`".res"` when `binary == true`, `".tres"` otherwise), and delegates to `ResourceSaver.save`. Passes `FLAG_COMPRESS` for binary saves; returns the raw `Error` integer (`0 == OK`).

**`load_from_disk(path) -> GameSaveData`**  
Builds a candidate list in priority order:
- If `path` already ends with `.tres` or `.res` → try that exact file first, then the alternate extension.
- Otherwise → try `<stem>.tres` then `<stem>.res`.

Uses `ResourceLoader.CACHE_MODE_IGNORE` so successive load calls in the same session always return freshly-deserialised instances (avoids stale cache surprises during round-trip tests).

**`compute_hash(data) -> String`**  
Feeds a `HashingContext` (SHA-256) with a deterministic byte stream:
```
<pos.x>|<pos.y>| <last_played>| <level0>|<level1>|… ||| <item0.id>|<qty>|<rarity>| …
```
The `|||` sentinel separates the level list from the inventory so no collision is possible between the two arrays. Returns a 64-character lowercase hex string.

---

### Why nested sub-resources round-trip losslessly
Godot's `ResourceFormatSaverText` / `ResourceFormatSaverBinary` automatically emit typed sub-resources inline when the nested type is registered via `class_name`. Because `ItemData` declares `class_name ItemData`, the engine knows how to reconstruct it on load without any extra flags. The typed `Array[ItemData]` hint on `inventory` reinforces this for both formats.
