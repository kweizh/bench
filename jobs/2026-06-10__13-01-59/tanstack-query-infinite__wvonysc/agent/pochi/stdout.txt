## How It Works

1. **`useInfiniteQuery`** is configured with:
   - `queryKey: ["feed"]` — caches feed data
   - `queryFn` — calls `fetchFeedPage(pageParam)` where `pageParam` is the cursor
   - `initialPageParam: 0` — starts at page 0
   - `getNextPageParam` — extracts `nextCursor` from the last page response; returns `null` when no more pages

2. **Page 1** (items 1–5) loads automatically on mount.
3. Clicking **"Load More"** calls `fetchNextPage()`, which fetches the next cursor from the API.
4. **Page 2** (items 6–10) is appended to the list alongside Page 1.
5. After Page 3 (items 11–15), the button disappears and "You have reached the end of the feed." is shown.
6. The button shows "Loading..." and is disabled while fetching.
