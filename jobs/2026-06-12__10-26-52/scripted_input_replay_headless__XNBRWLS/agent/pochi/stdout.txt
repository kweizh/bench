# libGDX Headless: Scripted Keystroke Replay Walker ## Background libGDX ships a non-rendering backend (`com.badlogicgames.gdx:gdx-backend-headless`) that swaps the live `Graphics`/`Input`/`Audio` implementations for mocks so that a libGDX `ApplicationListener` can be driven from a server, CI runner, or test harness. ## How it works

1. **`Main.java`** parses `--input=<path>`, reads the UTF-8 replay file line-by-line (skipping blank lines and `#` comments), maps each token case-insensitively to `Input.Keys.{UP,DOWN,LEFT,RIGHT}`, and exits non-zero with `Error: unknown key <token>` on any unrecognised token â€” all before the app boots.

2. **`HeadlessApplication`** is constructed with the `WalkerGame` listener and `updatesPerSecond=60`. The constructor sets `Gdx.input` to a default `MockInput`, then starts the main loop on a background thread.

3. **`WalkerGame.create()`** (called on the background thread before any `render()`) installs the `ScriptedInput` as `Gdx.input`, then calls `Gdx.input.setInputProcessor(...)` with an `InputAdapter` that updates `x`/`y` based on the keycode received.

4. **`ScriptedInput.tick()`** is called once per `render()` tick; it dispatches the next `keyDown`/`keyUp` pair to the registered `InputProcessor`. When the list is exhausted `isFinished()` returns `true`.

5. **`WalkerGame.render()`** calls `tick()` each frame. Once `isFinished()` is true it prints `Final position: (x, y)` exactly once and calls `Gdx.app.exit()`.

6. **`Main`** joins the `mainLoopThread` (accessed via reflection on the `protected` field) so the JVM waits for the simulation to complete.
