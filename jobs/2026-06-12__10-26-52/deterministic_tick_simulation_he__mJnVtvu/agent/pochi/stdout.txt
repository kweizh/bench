## Key design decisions

### `build.gradle`
- `application` plugin with `mainClass = 'com.example.sim.Main'` enables `./gradlew run --args="..."`.
- Three libGDX 1.14.2 dependencies: `gdx`, `gdx-backend-headless`, and `gdx-platform:natives-desktop` (the JNI layer needed by the headless loader).
- Sonatype releases/snapshots repos added alongside Maven Central (libGDX is published there).
- `options.encoding = 'UTF-8'` on all `JavaCompile` tasks suppresses encoding warnings.

### `SimulationApp.java` (`ApplicationAdapter`)
- **`create()`** вЂ” reads the config via `Gdx.files.absolute(configPath).read()` into `java.util.Properties`; if `ticks=0`, calls `Gdx.app.exit()` immediately so no `render()` ever fires.
- **`render()`** вЂ” performs exactly one Symplectic Euler step per call (velocity before position, `ax=0`, `ay=gravity_y`), increments `tickCount`, and calls `Gdx.app.exit()` once `tickCount == totalTicks`. No wall-clock time is used anywhere.
- **`dispose()`** вЂ” formats the output with `Locale.ROOT` and `%.6f`, replaces any stray `\r\n` with `\n` (LF-only), and writes via `OutputStreamWriter(FileOutputStream, UTF-8)`.

### `Main.java`
- `HeadlessApplicationConfiguration.updatesPerSecond = 0` вЂ” removes wall-clock throttling so the loop runs as fast as the CPU allows.
- After constructing `HeadlessApplication`, the main thread calls `findMainLoopThread()` which searches all thread groups for the thread named `"HeadlessApplication"` (the name libGDX assigns internally).
- `mainLoopThread.join()` blocks until the loop thread exits вЂ” which only happens after `pause()` + `dispose()` complete вЂ” guaranteeing the output file is fully flushed before the JVM exits naturally.
