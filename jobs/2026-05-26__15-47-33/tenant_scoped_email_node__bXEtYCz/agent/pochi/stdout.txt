## Implementation Details

The automation performs the following steps:

1. **Sets a tenant** using `@knocklabs/node` with:
   - ID: `tenant-{run-id}`
   - Name including run-id
   - `settings.branding.primary_color: '#3b82f6'`
   - Custom property `app_name: 'MyApp'`

2. **Upserts a workflow** using `@knocklabs/mgmt` in development environment with:
   - Key: `tenant-welcome-{run-id}`
   - Single channel step bound to `mailtrap` channel
   - Subject template: `Welcome to {{ tenant.app_name }}!`
   - HTML body template referencing `recipient.name` and `tenant.name`
   - Sender email: `sender-{run-id}@{MAILTRAP_DOMAIN}`

3. **Activates the workflow** in development environment

4. **Triggers the workflow** using `@knocklabs/node` with:
   - Tenant field set to `tenant-{run-id}`
   - Single inline recipient with ID `user-{run-id}`, email `{GMAIL_USER_NAME}+receiver-{run-id}@gmail.com`, and name including run-id

5. **Writes to output.log** with:
   - Tenant ID
   - Workflow Key
   - Workflow Run ID
   - Recipient Email
