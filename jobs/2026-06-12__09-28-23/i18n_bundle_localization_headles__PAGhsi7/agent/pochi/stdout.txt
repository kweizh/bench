# libGDX Headless: I18NBundle Localization Console ## Background libGDX ships an internationalization helper, `com.badlogic.gdx.utils.I18NBundle`, that loads a family of `.properties` files (a base file plus one per locale) and resolves keys for the active locale, falling back to the base bundle when a key is missing for the requested locale. Bundles are loaded from a `FileHandle` (typically `Gdx.files.internal("i18n/Messages")`), which makes the feature easy to drive under the non-rendering **headless** backend (`com.badlogicgames.gdx:gdx-backend-headless`) because no GPU, audio, or input plumbing is involved. In this task you will build a libGDX 1.14.2 project that boots a `HeadlessApplication`, ships a small family of localized resource bundles, and exposes a command-driven console that: 1. Switches between locales on demand. 2. Looks up plain keys with `bundle.get(...)`. 3. Formats parameterized keys with `bundle.format(...)`. 4. Demonstrates locale-to-base fallback for missing keys. 5. Reports a useful error to stderr (and exits non-zero) when the command file is malformed. ## Requirements * Create a Gradle (Java 17) project at `/home/user/gdx-game` that depends on libGDX `1.14.2` (`gdx`, `gdx-backend-headless`, `gdx-platform:natives-desktop`). * Implement a libGDX `ApplicationListener` (or `ApplicationAdapter`) that boots under `com.badlogic.gdx.backends.headless.HeadlessApplication`. * Provide three properties files under the project's `assets/i18n/` directory and load them via `Gdx.files.internal("i18n/Messages")`: * `assets/i18n/Messages.properties` (base / English content): ``` greeting=Hello farewell=Goodbye welcomeUser=Welcome, {0}! scoreReport=Player {0} scored {1} points gameName=Cosmic Quest ``` * `assets/i18n/Messages_fr.properties` (French; intentionally **omits** `gameName` to exercise base-bundle fallback): ``` greeting=Bonjour farewell=Au revoir welcomeUser=Bienvenue, {0} ! scoreReport=Le joueur {0} a marque {1} points ``` * `assets/i18n/Messages_de.properties` (German; also omits `gameName`): ``` greeting=Hallo farewell=Auf Wiedersehen welcomeUser=Willkommen, {0}! scoreReport=Spieler {0} hat {1} Punkte erzielt ``` Files must be UTF-8 encoded (the default for `I18NBundle.createBundle`). * Produce a runnable launcher (via the Gradle `application` plugin) at `/home/user/gdx-game/build/install/gdx-game/bin/gdx-game` that accepts a single `--input=` argument naming a command file and drives the headless app to completion. * Bundles must be loaded for each locale via `I18NBundle.createBundle(Gdx.files.internal("i18n/Messages"), new java.util.Locale(<code>))`. The base/default bundle (used for the `en` code and for fallback) must come from the same family. ### Command file format * UTF-8 text file, one command per line. * Lines starting with `#` are comments and skipped. * Blank lines are skipped. * Leading and trailing whitespace on each line is ignored. * Tokens within a line are separated by one or more spaces. * Supported commands (case-sensitive keywords): * `LOCALE <code>` — switch the active bundle to the locale identified by `<code>` (one of `en`, `fr`, `de`). Switching to `en` must use the base bundle family (i.e., either an explicit English locale or the base bundle directly so that the resolution behaves identically to looking up missing keys in the base file). * `GET <key>` — print the line `<key>=<value>` to stdout, where `<value>` is the result of `bundle.get(<key>)` on the active bundle. * `FORMAT <key> <arg1> [<arg2> ...]` — print the line `<key>=<value>` to stdout, where `<value>` is the result of `bundle.format(<key>, arg1, arg2, ...)` on the active bundle. Arguments are passed through as `String` values. ### Error handling The launcher must exit with a non-zero status code and not print any further `<key>=...` lines once an error has been emitted. Errors must go to **stderr** and use the following exact prefixes: * A `GET` or `FORMAT` issued before any `LOCALE` command: * `Error: no locale selected` * An unknown locale code in a `LOCALE` command: * `Error: unsupported locale <code>` (use the raw `<code>` token). * A `GET`/`FORMAT` against a key that exists in neither the active locale's file nor the base file: * `Error: missing key <key>` (use the raw `<key>` token). * An unknown command keyword (anything other than `LOCALE`, `GET`, `FORMAT`, comments, blank lines): * `Error: unknown command <token>` (use the raw `<token>` keyword). ## Implementation Hints * Pin `gdxVersion=1.14.2`.
