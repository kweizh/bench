## How it works

1. **`Main.java`** reads the input file from disk, then creates a `HeadlessApplication` with the `HeadlessPixmapRenderer` listener. The headless backend starts a non-daemon main loop thread, so the JVM stays alive until `Gdx.app.exit()` is called.

2. **`HeadlessPixmapRenderer.java`** extends `ApplicationAdapter`. In `create()` (running on the libGDX thread), it:
   - Parses the command script line-by-line, skipping blanks and `#` comments
   - `SIZE` allocates a `Pixmap` in `RGBA8888` format
   - `FILL`, `RECT`, `LINE`, `CIRCLE`, `PIXEL` convert 0–255 color values to `[0,1]` floats, call `setColor()`, then the appropriate `Pixmap` draw method
   - Writes the PNG via `PixmapIO.writePNG()` using `Gdx.files.absolute()`
   - Disposes the `Pixmap`, prints `RENDER_OK width=<w> height=<h> commands=<n>`, and calls `Gdx.app.exit()`

3. **`run.sh`** invokes `./gradlew --no-daemon --offline run --args="<input> <output>"` from the project root.

- Output format: `RENDER_OK width=200 height=200 commands=5` (regex `^RENDER_OK width=\d+ height=\d+ commands=\d+$`)
- Exit code 0 on success
- Blank lines and `#` comments are ignored; `SIZE` must be first
