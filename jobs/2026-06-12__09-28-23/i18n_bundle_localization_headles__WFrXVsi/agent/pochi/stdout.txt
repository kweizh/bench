# libGDX Headless: I18NBundle Localization Console ## Background libGDX ships an internationalization helper, `com.badlogic.gdx.utils.I18NBundle`, that loads a family of `.properties` files (a base file plus one per locale) and resolves keys for the active locale, falling back to the base bundle when a key is missing for the requested locale. Bundles are loaded from a `FileHandle` (typically `Gdx.files.internal("i18n/Messages")`), which makes the feature easy to drive under the non-rendering **headless** backend (`com.badlogicgames.gdx:gdx-backend-headless`) because no GPU, audio, or input plumbing is involved. In this task you will build a libGDX 1.14.2 project that boots a `HeadlessApplication`, ships a small family of localized resource bundles, and exposes a command-driven console that: 1. Switches between locales on demand. 2. Looks up plain keys with `bundle.get(...)`. 3. Formats parameterized keys with `bundle.format(...)`. 4. Demonstrates locale-to-base fallback for missing keys. 5. Reports a useful error to stderr (and exits non-zero) when the command file is malformed. ## Requirements * Create a Gradle (Java 17) project at `/home/user/gdx-game` that depends on libGDX `1.14.2` (`gdx`, `gdx-backend-headless`, `gdx-platform:natives-desktop`). * Implement a libGDX `ApplicationListener` (or `ApplicationAdapter`) that boots under `com.badlogic.gdx.backends.headless.HeadlessApplication`. * Provide three properties files under the project's `assets/i18n/` directory and load them via `Gdx.files.internal("i18n/Messages")`: * `assets/i18n/Messages.properties` (base / English content): ``` greeting=Hello farewell=Goodbye welcomeUser=Welcome, {0}! scoreReport=Player {0} scored {1} points gameName=Cosmic Quest ``` * `assets/i18n/Messages_fr.properties` (French; intentionally **omits** `gameName` to exercise base-bundle fallback): ``` greeting=Bonjour farewell=Au revoir welcomeUser=Bienvenue, {0} ! scoreReport=Le joueur {0} a marque {1} points ``` * `assets/i18n/Messages_de.properties` (German; also omits `gameName`): ``` greeting=Hallo farewell=Auf Wiedersehen welcomeUser=Willkommen, {0}! scoreReport=Spieler {0} hat {1} Punkte erzielt ``` Files must be UTF-8 encoded (the default for `I18NBundle.createBundle`). * Produce a runnable launcher (via the Gradle `application` plugin) at `/home/user/gdx-game/build/install/gdx-game/bin/gdx-game` that accepts a single `--input=` argument naming a command file and drives the headless app to completion. * Bundles must be loaded for each locale via `I18NBundle.createBundle(Gdx.files.internal("i18n/Messages"), new java.util.Locale(<code>))`. The base/default bundle (used for the `en` code and for fallback) must come from the same family. ### Command file format * UTF-8 text file, one command per line. * Lines starting with `#` are comments and skipped. * Blank lines are skipped. * Leading and trailing whitespace on each line is ignored. * Tokens within a line are separated by one or more spaces. * Supported commands (case-sensitive keywords): * `LOCALE <code>` — switch the active bundle to the locale identified by `<code>` (one of `en`, `fr`, `de`). Switching to `en` must use the base bundle family (i.e., either an explicit English locale or the base bundle directly so that the resolution behaves identically to looking up missing keys in the base file). * `GET <key>` — print the line `<key>=<value>` to stdout, where `<value>` is the result of `bundle.get(<key>)` on the active bundle. * `FORMAT <key> <arg1> [<arg2> ...]` — print the line `<key>=<value>` to stdout, where `<value>` is the result of `bundle.format(<key>, arg1, arg2, ...)` on the active bundle. Arguments are passed through as `String` values. ### Error handling The launcher must exit with a non-zero status code and not print any further `<key>=...` lines once an error has been emitted. Errors must go to **stderr** and use the following exact prefixes: * A `GET` or `FORMAT` issued before any `LOCALE` command: * `Error: no locale selected` * An unknown locale code in a `LOCALE` command: * `Error: unsupported locale <code>` (use the raw `<code>` token). * A `GET`/`FORMAT` against a key that exists in neither the active locale's file nor the base file: * `Error: missing key <key>` (use the raw `<key>` token). * An unknown command keyword (anything other than `LOCALE`, `GET`, `FORMAT`, comments, blank lines): * `Error: unknown command <token>` (use the raw `<token>` keyword). ## Implementation Hints * Pin `gdxVersion=1.14.2`. The Maven coordinates and Gradle wiring are summarised in the libGDX docs (https://libgdx.com/wiki/articles/dependency-management-with-gradle, https://libgdx.com/wiki/internationalization-and-localization). * The Gradle `application` plugin gives you a turnkey launcher: configure `mainClass` and run `./gradlew installDist` to produce `build/install/<projectName>/bin/<projectName>`. * Bootstrap the Gradle wrapper with `gradle wrapper --gradle-version 8.10 --distribution-type bin` so reproducible builds are possible from the project root. * `Gdx.files.internal("i18n/Messages")` resolves against the `assets/` root, so put your properties files at `assets/i18n/Messages*.properties` and make sure the `assets/` directory is on the Gradle distribution's classpath (the standard pattern is to add `assets/` as a Java/resources source directory or use `sourceSets.main.resources.srcDirs += ["assets"]`). * `HeadlessApplication` runs the listener on its own thread; block the launcher's `main` until the application has exited (for example by joining the headless main-loop thread) so the JVM stays alive long enough to flush the bundle lookups. * Do **not** touch `Gdx.gl*`; the headless backend has no GL context. * For locale-independent argument substitution stick to plain `{0}`, `{1}`, ... placeholders and pass the arguments as `String` values; this avoids locale-sensitive number/date formatting differences in `MessageFormat`. * `I18NBundle.createBundle` falls back to less-specific files automatically, so a French bundle missing the `gameName` key will read it from the base `Messages.properties` without any extra code. ## Acceptance Criteria * Project path: `/home/user/gdx-game`. * After running `./gradlew --no-daemon --quiet installDist` from the project root, the launcher script `/home/user/gdx-game/build/install/gdx-game/bin/gdx-game` must exist and be executable. * Command: `/home/user/gdx-game/build/install/gdx-game/bin/gdx-game --input=<path>` * `<path>` is the absolute path to a UTF-8 command file conforming to the format above. * Successful runs must: * Print one `<key>=<value>` line per `GET`/`FORMAT` command, in the order the commands appear in the file. * Print **nothing else** to stdout (no banner, prompt, debug log, or trailing summary). * Exit with status `0`. * Runs containing a malformed command must: * Print exactly one matching `Error: ...` line to **stderr**, using the prefixes documented above. * Exit with a **non-zero** status code. * Not emit any further `<key>=<value>` lines on stdout after the error. * An empty command file (no non-blank, non-comment lines) must produce no stdout output and exit `0`. * The application must boot under `com.badlogic.gdx.backends.headless.HeadlessApplication` and load each bundle via `com.badlogic.gdx.utils.I18NBundle.createBundle(FileHandle, Locale)` against `Gdx.files.internal("i18n/Messages")`. (This is verified behaviourally through the locale fallback test, which depends on `I18NBundle`'s fallback rules to resolve `gameName` from the base bundle when the active locale is `fr` or `de`.)
