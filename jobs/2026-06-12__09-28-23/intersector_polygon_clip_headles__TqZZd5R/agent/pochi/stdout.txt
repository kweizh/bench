# libGDX Headless Polygon Geometry CLI ## Background You are building a deterministic command-line geometry utility on top of libGDX's headless backend. The tool reads a small text-based scripting language describing 2D convex polygons and queries, then evaluates each query using libGDX math primitives (notably `com.badlogic.gdx.math.Intersector` and `com.badlogic.gdx.math.Polygon`) inside a `HeadlessApplication` and writes one result per query to standard output. No OpenGL context, no native windowing, and no audio must be initialized. ## Requirements * Implement a Gradle-based libGDX project at `/home/user/myproject` that runs entirely with the headless backend (`com.badlogicgames.gdx:gdx-backend-headless:1.14.2`). * Pin the libGDX version to `1.14.2` (core `com.badlogicgames.gdx:gdx:1.14.2`). * Wire an `ApplicationListener` (or `ApplicationAdapter`) that runs under `new HeadlessApplication(listener, config)` with `config.updatesPerSecond = 0` so the program ticks once and exits. * Read the path to the script file from the first program argument (forwarded via `--args`). * For each non-empty, non-comment line in the script, execute exactly one of the commands below and print exactly one output line to stdout. Use libGDX's `Intersector` and `Polygon` classes for the geometric primitives. * After processing every line, call `Gdx.app.exit()` and join the main loop thread cleanly. * A `run.sh` wrapper script must be provided at the project root that accepts the script-file path as the first argument and forwards it to the headless launcher, writing only the program's stdout lines (no Gradle banner / progress output) to stdout. * A line starting with `#` or a blank line is ignored. ### Script language All coordinates are floating point numbers. Polygon names match `[A-Za-z][A-Za-z0-9_]*`. Polygons defined by the script are convex and given in counter-clockwise order. * `POLY <name> <x1> <y1> <x2> <y2> ... <xn> <yn>` — store a polygon with the given vertex coordinates (at least 3 vertices, so at least 6 numbers). * Output: `POLY <name> <vertexCount>` * `OVERLAP <a> <b>` — test whether convex polygons `<a>` and `<b>` overlap using `Intersector.overlapConvexPolygons`. * Output: `OVERLAP <a> <b> true` or `OVERLAP <a> <b> false` * `CONTAINS <name> <x> <y>` — test whether point `(x, y)` is inside polygon `<name>` using `Intersector.isPointInPolygon` on the stored vertex array. * Output: `CONTAINS <name> <x> <y> true` or `CONTAINS <name> <x> <y> false` * Print the input coordinates back exactly as they appeared in the script line (preserve the formatting tokens the user typed). * `SEGMENTS <ax> <ay> <bx> <by> <cx> <cy> <dx> <dy>` — intersect segment `AB` with segment `CD` using `Intersector.intersectSegments(Vector2, Vector2, Vector2, Vector2, Vector2)`. * If the segments meet, output: `SEGMENTS hit <px> <py>` where `<px>` and `<py>` are the intersection point coordinates rounded to **three decimal places** using `Locale.ROOT` (use `String.format(Locale.ROOT, "%.3f", value)`). * Otherwise, output: `SEGMENTS miss` * `AREA <name>` — compute the signed area of polygon `<name>` via `Polygon.area()`. * Output: `AREA <name> <value>` where `<value>` is rounded to three decimal places using `Locale.ROOT`. An unknown command, a reference to an undefined polygon, or a malformed numeric token must cause the program to print `ERROR <line-number> <command-or-reason>` (1-based line number counting blank and comment lines) and continue to the next line. ## Implementation Hints * The `gdx-backend-headless` artifact pulls in the desktop natives loader, so a plain Gradle `application` plugin run is sufficient — no LWJGL window is required. * `Intersector.overlapConvexPolygons` accepts `com.badlogic.gdx.math.Polygon` instances. `Intersector.isPointInPolygon` takes a `float[]` (or `(float[] polygon, int offset, int count, float x, float y)`). * `HeadlessApplicationConfiguration.updatesPerSecond = 0` runs the loop as fast as possible; you can do all your work in `create()` and then call `Gdx.app.exit()`. * Read the script through `Gdx.files.absolute(path)` or plain `java.nio.file.Files` — both work under the headless `Files` implementation. * Make sure your `run.sh` filters out Gradle's own logging (e.g. use `./gradlew -q` or capture only the launcher's stdout) so that the only lines emitted are the result lines defined above. * The headless launcher runs the `ApplicationListener` on its own thread;
