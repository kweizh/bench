# Interpolation Curve Sampler with libGDX Headless Backend ## Background libGDX ships a tweening utility, `com.badlogic.gdx.math.Interpolation`, that exposes named easing curves (`linear`, `smooth`, `smoother`, and many more) for animating values between two endpoints. The class is part of `com.badlogicgames.gdx:gdx` and does not require an OpenGL context, so it can be driven by the headless backend (`com.badlogicgames.gdx:gdx-backend-headless`) and used inside CI / server pipelines. Build a small, fully-deterministic Java application that drives the libGDX `Interpolation` API from inside an `ApplicationListener` running under `HeadlessApplication`, samples a named easing curve over the unit interval, and writes the resulting samples to a file. ## Requirements * Create a Gradle (single-module) Java project at `/home/user/gdx-interp`. * Use libGDX `1.14.2` with the headless backend (`com.badlogicgames.gdx:gdx-backend-headless:1.14.2`) on top of `com.badlogicgames.gdx:gdx:1.14.2` and the `com.badlogicgames.gdx:gdx-platform:1.14.2:natives-desktop` natives classifier. * Implement an `ApplicationListener` (or `ApplicationAdapter`) whose `create()` reads a Java properties config file via `Gdx.files.absolute(...)`, whose `render()` produces exactly one sample per tick, and whose `dispose()` writes the final result to an output file. * Bootstrap the application from a `main` method that constructs a `HeadlessApplication` with a `HeadlessApplicationConfiguration`. After the last sample call `Gdx.app.exit()` and join the main-loop thread so the JVM exits cleanly. * The sample at index `i` (zero-based) must equal `Interpolation.<curve>.apply(start, end, t)` where `t = i / (samples - 1)` as a single-precision `float` and `samples >= 2`. * Supported curve names (must be looked up on `com.badlogic.gdx.math.Interpolation`): `linear`, `smooth`, `smoother`. The program must reject any other curve name with a non-zero exit code and an error message on stderr. * Make the program runnable through Gradle's `application` plugin so it can be launched with `./gradlew run --args="<config-path> <output-path>"`. ## Implementation Hints * Look at `HeadlessApplication` and `HeadlessApplicationConfiguration`. Setting `updatesPerSecond = 0` lets the main loop run as fast as possible without wall-clock pacing, which is what you want for a finite, deterministic sweep. * `MockGraphics` in the headless backend cannot run OpenGL code, so do NOT call anything under `Gdx.gl*`; keep the listener pure logic. * `Gdx.app.exit()` is asynchronous (it posts a runnable that flips a `running` flag and triggers `pause()` + `dispose()`). To guarantee the output file is flushed before the JVM exits, write the file from `dispose()` and `join()` the main-loop thread from `main`. * Use `java.util.Properties` (or any other parser) to read the config; access it through `Gdx.files.absolute(path).read()` so the libGDX `FileHandle` abstraction is exercised. * All numeric values written to the output file MUST use `Locale.ROOT` (or equivalent) when formatting, so the decimal separator is `.`. * The sample at `i = samples - 1` must use `t = 1.0f` exactly (do not let floating-point drift produce `0.9999...`). After exactly `samples` ticks, the program must call `Gdx.app.exit()`. No additional samples may be produced after `samples` ticks. * Unknown `curve` value: the program must NOT create or overwrite `<output-path>` and must exit with a non-zero status code. * The program must terminate within 60 seconds for any input with `samples <= 100000`.
